流处理简介

" 将数据库翻转 " 是数据工程领域的一篇具有影响力的文章，促成了 Kafka 的创建。此后，像 Redpanda 和 Redis Streams 这样的实现相继出现，推动了实时数据处理生态系统的发展。

与事件驱动编程的对比​

与事件驱动编程类似，流处理是一种编程范式，旨在近实时地处理事件或尽快处理事件。区分这两者的一种方法可能是频率。流是高吞吐量的连续事件序列：与许多短暂连接相比，您只需保持一个连接处于打开状态，等待事件的到来。

与批处理的对比​

流处理可以被视为具有极小批量大小的批处理。批量大小并不一定非得是一个。根据您的延迟要求和处理能力，消息可以以毫秒或秒为单位进行微批处理。但是，“该变化可能需要长达 24 小时才能反映”的时代已经过去了！

为什么选择 Rust？​

我们希望构建最佳的流处理平台，让 Rust 的独特特性真正发挥作用：

多线程异步​

与其他语言不同，Rust 的异步执行是多线程的。它允许您根据需要扩展进程，使用尽可能多的线程以充分利用 CPU，实现最大并发性。您无需设置外部队列系统！

可预测的延迟​

作为一种没有垃圾回收的语言，Rust 没有随机的时刻会触发垃圾回收器并导致抖动。当您拥有长管道时，这些抖动往往会传播并在下游放大。然而，Rust 并不是自动低延迟的 - 您仍然需要花费相当的精力进行优化。但您将拥有一个良好的起点。

自包含​

与其他语言不同，推荐的 Rust 程序打包方式是将所有内容静态链接到一个可执行文件中 - 通常只有几兆字节大小。并且不需要安装或预热 - 它立即启动，这对流处理而言是一个额外的好处。

低资源使用​

与其他编译语言类似，Rust 使用的内存比基于虚拟机的语言显著更少。而且由于无需 JIT，Rust 的 CPU 开销也更小。

长时间运行的安全性​

同样，由于没有垃圾回收，Rust 程序不那么容易受到“在数天内缓慢内存膨胀”的影响（从技术上讲，这不是泄漏）。因此，更少出现内存溢出崩溃的风险，您不必“每周重启一次进程”。尽管如此，您仍需对堆分配保持谨慎。

生态系统​

最后，Rust 拥有出色的异步编程库生态系统：基于异步 IO 的网络库、无锁通道和其他数据结构，使异步编程更具可用性和趣味性。

废话不多说，让我们开始吧！
