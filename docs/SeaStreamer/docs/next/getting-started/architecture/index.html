SeaStreamer 架构

sea-streamer 的架构由多个子库构成：

  • sea-streamer-types
  • sea-streamer-socket
    • sea-streamer-kafka
    • sea-streamer-redis
    • sea-streamer-stdio
    • sea-streamer-file
  • sea-streamer-runtime

sea-streamer-types: 特征与类型​

该库定义了 SeaStreamer API 的所有特征和类型，但不提供具体实现。

sea-streamer-socket: 后端无关的 Socket API​

与 SeaORM 允许您为不同的数据库构建应用程序类似，SeaStreamer 允许您为不同的流处理服务器构建流处理器。

虽然 sea-streamer-types 库提供了良好的基于特征的抽象，但该库提供了具体类型的 API，以便您的程序可以在运行时从任何用户选择的 SeaStreamer 后端进行流式输入/输出。

这使您能够做一些很酷的事情，比如在本地生成数据，然后将其流式传输到 Redis / Kafka。反之亦然，可以从服务器获取数据并在本地处理。所有这些 无需重新编译 流处理器。

sea-streamer-kafka: Kafka / Redpanda 后端​

这是 SeaStreamer 的 Kafka / Redpanda 后端实现。该库提供了一个全面的类型系统，使得与 Kafka 的交互更加简便和安全。

所有 API（许多是同步的）都被正确地封装为异步的。方法还标记为 &mut 以消除潜在的竞争条件。

该库依赖于 rdkafka，而 rdkafka 又依赖于 librdkafka-sys，后者是 librdkafka 的包装器。

sea-streamer-redis: Redis 后端​

这是 SeaStreamer 的 Redis 后端实现。该库在 Redis 之上提供了高层次的异步 API，使与 Redis Streams 的交互变得简单可靠：

  • 实现了熟悉的 SeaStreamer 抽象接口
  • 提供全面的类型系统，以指导/限制 API 的使用
  • 高层次 API，您不再需要调用 XADD、XREAD 或 XACK
  • 无需互斥锁的实现：通过消息传递实现并发

该库构建于 redis 之上。

sea-streamer-stdio: 标准输入/输出后端​

这是 SeaStreamer 的 stdio 后端实现。它可以与 UNIX 管道连接在一起，为流处理器或本地数据处理提供极大的灵活性。

您可以通过管道将处理器连接在一起：processor_a | processor_b。

sea-streamer-file: 文件后端​

这与 sea-streamer-stdio 非常相似，但不同之处在于 Stdio 实时工作，而 sea-streamer-file 支持实时和重放。这意味着，File 具备遍历 .ss（sea-stream）文件并查找/回放到特定时间戳/偏移量的能力。

此外，Stdio 只能处理 UTF-8 文本数据，而 File 能够处理二进制数据。

sea-streamer-runtime: 异步运行时抽象​

该库提供了一小组函数，以对齐 async-std 和 tokio 之间的类型签名，使您能够构建针对这两种运行时的通用应用程序。
