消费者

Consumer 特性定义了流消费者通用的接口。

已实现：

  • KafkaConsumer
  • RedisConsumer
  • StdioConsumer
  • FileConsumer

ConsumerOptions​

ConsumerMode​

有三种模式：

RealTime​

这是“原味”的流消费者。它不会自动提交，因此只从现在开始消费消息。

Resumable​

当进程重启时，它将从上次提交的序列继续流。

信息

Redis / Kafka 语义​

它将使用对这台主机唯一的组ID：在物理机器上，它将使用MAC地址。 在Docker容器内，它将使用容器ID。

信息

Redis 语义​

Redis要求消费者自行分配消费者ID。如果未设置，SeaStreamer会使用主机ID + 进程ID + 线程ID + 时间戳的组合。

警告

Stdio / File 语义​

目前不支持。

LoadBalanced​

你应该手动分配一个消费者组。负载均衡机制是特定于实现的。

ConsumerGroup​

消费者组是一个字符串，客户端用它向流服务器识别自己。这样当你重新连接时，可以从服务器下载状态。从代理的角度看，这就是全部。客户端可以从任何主机或网络连接。

多个消费者可以共享同一个消费者组，并同时保持与服务器的连接。通常，目的是为了实现负载均衡。精确的语义取决于后端。

信息

Kafka 语义​

如果多个消费者共享同一个组，则只有该组中的一个消费者会收到一条消息，即它是负载均衡的。

然而，负载均衡机制正是让Kafka与众不同的地方：

每个流被分成多个分片（称为分区），并且每个分区只会被分配给组中的一个消费者。

假设有两个消费者（在组中）和两个分区，那么每个消费者将接收来自一个分区的消息，因此它们是负载均衡的。

如果有两个消费者和三个分区，那么一个消费者将被分配两个分区，而另一个只会被分配一个。

但是，如果流只有一个分区，即使有许多消费者，这些消息也只会被分配的消费者接收，其他消费者将处于待机模式，从而导致热故障转移设置。

信息

Redis 语义​

同一组中的多个消费者共享相同的流。这是以先到先得的方式负载均衡的。这可以被视为动态负载均衡：更快的消费者会消费更多的消息。

结果，必须按每条消息进行ack。在SeaStreamer中，这变成了两步：ack是非阻塞的，它会在内部缓冲acks，并定期或根据你的请求commit到Redis。有多种自动ack / commit机制可供选择：Immediate、Delayed、Rolling和Disabled。

SeaStreamer还实现了自动故障转移，其中其他消费者遗留的消息可以在设定的时间后被“申领”，假设它们已经死亡。这可以通过自动申领选项来配置。

信息

Stdio / File 语义​

如果多个消费者共享同一个组，则只有组中的一个会接收一条消息。 这是以轮询方式负载均衡的。

next​

轮询并接收一条消息：它会等待直到有新消息。 此方法可从多个线程调用。

stream​

返回一个实现了Stream Trait的异步流。你不能从同一个消费者创建多个流，也不能在流式传输时执行任何操作。

它允许你做些整洁的事情：

let items = consumer
    .stream()
    .take(num)
    .map(process_message)
    .collect::<Vec<_>>()
    .await

assign​

将此消费者分配给特定的分片。可以多次调用来分配到多个分片。你不能分配尚未订阅的流。

它仅在下一个Consumer::seek或Consumer::rewind时生效。

unassign​

取消分配一个分片。如果此消费者未被分配到此流或分片，将返回ConsumerNotAssigned。

rewind​

将流回绕到特定的序列号。

信息

Kafka 语义​

如果消费者尚未分配，将使用分片ZERO。这个异步方法不是取消安全的。你必须等待这个未来，且在完成之前这个Consumer将无法用于任何操作。

信息

Redis 语义​

在Redis中，序列号包含一个时间戳，所以回绕几乎与查找相同，但更精确：你可以回绕到毫秒内的某个点。

信息

文件语义​

影响所有流。 如果消费者订阅了多个流，它将通过第一个流键查找。 它撤销了消费者的组成员资格。

警告

Stdio 语义​

这没有被Stdio后端实现。

seek​

将所有流查找至给定的时间点。它将从比to之后的最早消息开始消费。

信息

Kafka 语义​

这将自我分配所有分片。这个异步方法不是取消安全的。你必须等待这个未来，且在完成之前这个Consumer将无法用于任何操作。

信息

Redis 语义​

查找一个消费者会将其从消费者组中分离，如果它已经被分配了一个。它实际上使其成为一个实时消费者。

信息

文件语义​

影响所有流。 如果消费者订阅了多个流，它将通过第一个流键查找。 它撤销了消费者的组成员资格。

警告

Stdio 语义​

这没有被Stdio后端实现。

自动流重置​

信息

Kafka 语义​

AutoOffsetReset选项允许如果消费者组未分配，或者消费者组没有先前状态，则将流重置为Earliest。默认为Latest。

信息

Redis 语义​

AutoStreamReset选项允许如果消费者组未分配，或者消费者组没有先前状态，则将流重置为Earliest。默认为Latest。

信息

文件语义​

AutoStreamReset选项允许在流启动时将流重置为Earliest。默认为Latest。 从同一文件流式传输的多个消费者如果都是Latest，则会共享相同的文件句柄。 在消费者上调用seek/rewind会使其脱离消费者组，并将打开一个新的文件句柄。

警告

Stdio 语义​

这没有被Stdio后端实现。
