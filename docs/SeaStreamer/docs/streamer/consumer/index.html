Consumer

Consumer 特性定义了流消费者的公共接口。

实现者包括：

  • KafkaConsumer
  • RedisConsumer
  • StdioConsumer
  • FileConsumer

ConsumerOptions​

ConsumerMode​

有 3 种模式：

RealTime​

这是 “原始” 的流消费者。它不会自动提交，因此仅从现在开始消费消息。

Resumable​

当进程重启时，它将从之前已提交的序列继续流。

info

Redis / Kafka 语义​

它将使用该主机唯一的组 ID：在物理机上，它将使用 MAC 地址。 在 Docker 容器内，它将使用容器 ID。

info

Redis 语义​

Redis 要求消费者自我分配消费者 ID。如果未设置，SeaStreamer 将使用 host id + process id + thread id + timestamp 的组合。

caution

Stdio / File 语义​

当前不支持。

LoadBalanced​

您应手动分配一个消费者组。负载均衡机制是实现特定的。

ConsumerGroup​

消费者组是一个字符串，供客户端在流服务器上识别自己。这样，当您重新连接时，可以从服务器下载状态。从代理商的角度来看，这一点至关重要。客户端可以从任何主机或网络连接。

多个消费者可以共享同一消费者组，并同时保持与服务器的连接。通常，目的是实现负载均衡。精确的语义是后端特定的。

info

Kafka 语义​

如果多个消费者共享同一组，则组中的仅一个消费者会接收消息，即它是负载均衡的。

然而，负载均衡机制是 Kafka 的不同之处：

每个流分为多个分片（称为分区），每个分区将仅分配给组中的一个消费者。

假设有 2 个消费者（在该组内）和 2 个分区，则每个消费者将从一个分区接收消息，因此它们是负载均衡的。

如果有 2 个消费者和 3 个分区，则一个消费者将被分配 2 个分区，而另一个将仅被分配 1 个。

但是如果流只有 1 个分区，即使有多个消费者，这些消息也仅会被分配的消费者接收，其他消费者将处于待命模式，导致热备份设置。

info

Redis 语义​

同一组中的多个消费者共享同一流。这是按先到先服务的方式进行负载均衡。这可以视为动态负载均衡：更快的消费者将消费更多消息。

因此，ack 必须针对每条消息进行。它在 SeaStreamer 中变成两个步骤，ack 和 commit：ack 是非阻塞的，它将在内部缓冲 ack，并在定期时间间隔内或根据您的请求提交到 Redis。有多种 自动 ack / commit 机制 可供选择：Immediate, Delayed, Rolling 和 Disabled。

SeaStreamer 还实现了自动故障转移，在假定其他消费者死亡后，可以在设定时间段后“认领”剩余消息。这可以通过 auto claim 选项进行配置。

info

Stdio / File 语义​

如果多个消费者共享同一组，则组中的仅一个会接收消息。 这是以轮询的方式进行负载均衡。

next​

轮询并接收一条消息：它会等待直到有新消息。 此方法可以从多个线程调用。

stream​

返回一个实现 Stream Trait 的异步流。您不能从同一消费者创建多个流，也不能在流播过程中执行任何操作。

它允许您进行一些很酷的事情：

let items = consumer
    .stream()
    .take(num)
    .map(process_message)
    .collect::<Vec<_>>()
    .await

assign​

将此消费者分配给特定的分片。可以多次调用以分配 给多个分片。您不能分配尚未订阅的流。

它将在下一个 Consumer::seek 或 Consumer::rewind 中生效。

unassign​

取消分配一个分片。如果该消费者尚未分配给此流或分片，则返回 ConsumerNotAssigned。

rewind​

将流回溯到特定序列号。

info

Kafka 语义​

如果消费者尚未分配，则将使用分片 0。此异步方法不支持取消安全。您必须等待此未来，并且在完成之前，此消费者将无法用于任何操作。

info

Redis 语义​

在 Redis 中，序列号由时间戳组成，因此回溯几乎与查找相同，但更精确：您可以回溯到毫秒内的特定点。

info

File 语义​

影响所有流。 如果消费者订阅多个流，则将根据第一个流键进行查找。 它撤销消费者的组成员资格。

caution

Stdio 语义​

这在 Stdio 后端中未实现。

seek​

在给定的时间点查找所有流。它将开始从时间戳晚于 to 的最早消息中消费。

info

Kafka 语义​

这将自我分配所有分片。此异步方法不支持取消安全。您必须等待此未来，并且在完成之前，此消费者将无法用于任何操作。

info

Redis 语义​

查找一个消费者将使其与消费者组脱离关系（如果已分配）。这有效地使它成为一个实时消费者。

info

File 语义​

影响所有流。 如果消费者订阅多个流，则将根据第一个流键进行查找。 它撤销消费者的组成员资格。

caution

Stdio 语义​

这在 Stdio 后端中未实现。

自动流重置​

info

Kafka 语义​

AutoOffsetReset 选项允许在未分配消费者组时将流重置为 Earliest，或者消费者组没有先前状态时进行重置。默认值为 Latest。

info

Redis 语义​

AutoStreamReset 选项允许在未分配消费者组时将流重置为 Earliest，或者消费者组没有先前状态时进行重置。默认值为 Latest。

info

File 语义​

AutoStreamReset 选项允许在流启动时将流重置为 Earliest。默认值为 Latest。 多个消费者从同一文件流时，如果他们都设置为 Latest，将共享同一文件句柄。 在消费者上调用查找/回溯将使其与消费者组脱离，并将打开一个新的文件句柄。

caution

Stdio 语义​

这在 Stdio 后端中未实现。
