SeaStreamer 架构

sea-streamer 的架构由多个子库构成：

  • sea-streamer-types
  • sea-streamer-socket
    • sea-streamer-kafka
    • sea-streamer-stdio
  • sea-streamer-runtime

sea-streamer-types: 特性和类型​

该库定义了 SeaStreamer API 的所有特性和类型，但不提供任何实现。

sea-streamer-socket:后端无关的 Socket API​

类似于 SeaORM 允许您为不同的数据库构建应用程序，SeaStreamer 允许您为不同的流服务器构建流处理器。

虽然 sea-streamer-types 库提供了很好的基于特性的抽象，但这个库提供了具体类型的 API， 使得您的程序可以 在运行时 从任何用户选择的 SeaStreamer 后端进行流式传输。

这让您可以做一些很酷的事情，比如在本地生成数据然后将其流式传输到 Kafka。或者反过来， 从 Kafka 中获取数据并在本地处理它。所有这些 无需重新编译 流处理器。

sea-streamer-kafka: Kafka / Redpanda 后端​

这是 SeaStreamer 的 Kafka / Redpanda 后端实现。 该库提供了一个全面的类型系统，使与 Kafka 的交互变得更简单和安全。

该库依赖于 rdkafka， 而 rdkafka 又依赖于 librdkafka-sys， 后者是 librdkafka 的一个封装。

sea-streamer-stdio: 标准 I/O 后端​

这是 SeaStreamer 的 stdio 后端实现。它旨在通过 unix 管道连接在一起，为开发流处理器或本地处理数据提供极大的灵活性。

您可以通过管道将处理器连接在一起： processor_a | processor_b。

sea-streamer-runtime: 异步运行时抽象​

该库提供了一小组函数，使 async-std 和 tokio 之间的类型签名保持一致， 以便您可以构建适用于这两种运行时的通用应用程序。
