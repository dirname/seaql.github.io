流处理简介

"将数据库内部翻转" 是数据工程领域的一篇具有影响力的文章，促成了 Kafka 的成立。从那时起，如 Redpanda 和 Redis Streams 等实现相继出现，推动了实时数据处理生态系统的发展。

与事件驱动编程的对比​

流处理类似于事件驱动编程，都是一种编程范式，旨在以接近实时的方式处理事件或在事件发生后尽快处理。将两者区分的一种方法可能是频率。流是一系列连续的事件，具有高吞吐量：与许多短暂的连接不同，你只需保持一个连接处于打开状态。

与分析处理的对比​

假设我们想计算某个属性在特定时间段的平均值。OLAP 数据库允许我们在一张包含数百万行的大表上高效地计算这一值。要在流处理上实现这一点，我们可以重放流并通过流处理器处理它，这可能被认为是较慢的。一旦处理器稳定运行，输出实际上可以以极小的延迟实时更新。

为什么选择 Rust？​

我们希望构建一个最佳的流处理平台，在这里，Rust 的独特特性得以充分展现：

多线程异步​

与其他语言不同，Rust 的异步执行是多线程的。它允许你根据需要扩展进程，使用尽可能多的线程充分利用 CPU，以实现最大并发性。

可预测的延迟​

作为一种没有垃圾回收的语言，没有随机的时间点会造成垃圾回收器启动，从而导致抖动。当你有一个长管道时，这些抖动往往会传播并在下游放大。然而，Rust 并不是自动具有低延迟的 - 你仍需花费相当的精力进行优化。但你将拥有一个良好的起点。

自包含​

与其他语言不同，Rust 程序的推荐打包方式是将所有内容静态链接到一个可执行文件中 - 通常只有几兆字节大小。而且不需要安装或预热 - 它可以立即启动，这对流处理来说是一个额外的好处。

低资源使用​

像其他编译语言一样，Rust 的内存使用量远低于基于虚拟机的语言。并且无需 JIT，Rust 也具有更少的 CPU 开销。

长期安全​

同样，由于没有垃圾回收，Rust 程序不太容易出现“在几天内缓慢增长的内存膨胀”（从技术上讲，这并不是内存泄漏）。出现内存不足崩溃的风险较小，因此你不必“每星期重启进程”。不过，你仍需对堆分配保持谨慎。

生态系统​

最后，Rust 拥有优秀的异步编程库生态系统：基于异步 IO 的网络库、无锁通道以及其他数据结构使异步编程变得更为人性化和有趣。

不再多说，让我们开始吧！
