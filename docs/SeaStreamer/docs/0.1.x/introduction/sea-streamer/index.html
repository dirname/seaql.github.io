SeaStreamer 概念

Streamer​

流媒体服务器。它通过一个 URI 进行标识，所有生产者和消费者都可以连接到这个 URI。

Cluster​

流媒体服务器被认为是一个集群：它可以在多个节点之间水平扩展。

Stream​

一个流由一系列共享相同关键字（在 Kafka 中称为 topic）的消息组成。每条消息都有一个时间戳、序列号（在 Kafka 中称为 offset）、分片 ID（在 Kafka 中称为 partition number）以及负载。每条消息通过（流关键字、分片 ID、序列号）元组唯一标识。

Stream URL​

在 SeaStreamer 中，流是资源，可以通过包含（协议、主机、流）的 URL 进行访问。一个示例流 URL 是 kafka://streamer.sea-ql.org:12345/my_stream。

Consumer​

流消费者订阅一个或多个流，并从集群中的一个或多个节点接收消息。

消费者可以将流回滚到任意点（通过时间戳或序列号定位），并继续流式传输。

Consumer Mode​

有两种消费模式：

  1. 实时：我们只关心最新的消息，错过旧数据也无妨
  2. 可恢复：当消费者重新订阅时，它将从最后消费的消息继续

Producer​

流生产者向流媒体服务器发送消息，服务器将在集群中存储这些消息，并将它们传递给客户端。

生产者可以使用任何流关键字发送消息，但在 SeaStreamer 中，我们建议您将每个生产者固定到特定的流关键字。

Processor​

流处理器同时是消费者和生产者。它消费消息，转换它们并生成另一个流。

SeaStreamer 旨在使开发和操作流处理器变得简单灵活。

Stream Semantics​

先进的概念，如分片、负载均衡和事务是特定于后端的，您应该阅读相关的流媒体后端文档。
