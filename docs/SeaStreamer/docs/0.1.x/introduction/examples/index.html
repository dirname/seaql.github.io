示例处理器

我们维护了一组 示例，您可以将其作为开发流处理器的起点。

这些示例适用于 tokio 和 async-std，并使用 sea-streamer-socket API：

  • consumer：一个基本的消费者
  • producer：一个基本的生产者
  • processor：一个基本的流处理器
  • resumable：一个可恢复的流处理器，能够从中断处继续
  • buffered：一个具有内部缓冲和批处理的高级流处理器
  • blocking：一个用于处理阻塞 / CPU 密集型任务的高级流处理器

运行基本处理器示例​

在 Kafka 下：

# 生成一些输入
cargo run --bin producer -- --stream kafka://localhost:9092/hello1 &
# 启动处理器，生成一些输出
cargo run --bin processor -- --input kafka://localhost:9092/hello1 --output kafka://localhost:9092/hello2 &
# 重放输出
cargo run --bin consumer -- --stream kafka://localhost:9092/hello2
# 记得停止进程
kill %1 %2

在 Stdio 下：

# 将生产者的输出管道连接到处理器
cargo run --bin producer -- --stream stdio:///hello1 | \
cargo run --bin processor -- --input stdio:///hello1 --output stdio:///hello2

运行可恢复处理器示例​

可恢复处理器可以在任何时间被终止，并将继续从中断处继续处理。 这通常被称为 "至少一次" 处理，意味着不会丢失任何消息，但同一条消息可能被处理两次。

# 生成大量输入
cargo run --bin producer -- --stream kafka://localhost:9092/hello1
# 运行处理器，但在处理整个流之前终止它
cargo run --bin resumable -- --input kafka://localhost:9092/hello1 --output stdio:///hello2 | head -n 10
cargo run --bin resumable -- --input kafka://localhost:9092/hello1 --output stdio:///hello2 | head -n 10
cargo run --bin resumable -- --input kafka://localhost:9092/hello1 --output stdio:///hello2 | head -n 10

输出：

运行缓冲处理器示例​

时钟的速度是处理器的 10 倍，因此我们期望每个批次大约包含 10 条消息。

当输入流的频率很高，但处理器的阻抗很高时，这种模式非常有用。

更深入的技术讨论：
例如，为了将记录插入数据库，批量插入更有效。但您不能简单地将批量大小固定为 10 或 100，因为它可能缓冲了 9 条消息并在等待第 10 条，并且您无法处理突发的消息。

那么，如何最小化整体任务执行时间呢？您可以解耦这两个繁忙循环，并使用队列将它们松散连接：现在两个循环可以以最佳频率运转，最大化处理器的整体吞吐量。

alias clock='cargo run --package sea-streamer-stdio --features=executables --bin clock'
clock -- --stream clock --interval 100ms | \
cargo run --bin buffered -- --input stdio:///clock --output stdio:///output

输出：

[2023-02-27T10:43:58 | output | 0] [batch 0] { "tick": 0 } 处理完成
[2023-02-27T10:43:59 | output | 1] [batch 1] { "tick": 1 } 处理完成
[2023-02-27T10:43:59 | output | 2] [batch 1] { "tick": 2 } 处理完成
[2023-02-27T10:43:59 | output | 3] [batch 1] { "tick": 3 } 处理完成
[2023-02-27T10:43:59 | output | 4] [batch 1] { "tick": 4 } 处理完成
[2023-02-27T10:43:59 | output | 5] [batch 1] { "tick": 5 } 处理完成

运行阻塞处理器示例​

时钟的速度是处理器的 3 倍，但我们有 4 个线程，因此我们期望它能够实时赶上。任务随机分配给线程，采用 "扇出" 模式。

这种模式在您需要执行阻塞 IO 或 CPU 密集型计算时非常有用。

alias clock='cargo run --package sea-streamer-stdio --features=executables --bin clock'
clock -- --stream clock --interval 333ms | \
cargo run --bin blocking -- --input stdio:///clock --output stdio:///output

输出：

[2023-03-07T06:00:53 | output | 1] [thread 1] { "tick": 1 } 处理完成
[2023-03-07T06:00:53 | output | 2] [thread 2] { "tick": 2 } 处理完成
