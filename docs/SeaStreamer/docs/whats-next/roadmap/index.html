Version: 0.3.x

路线图

感谢您阅读到本章节的文档。如果您发现该项目有趣和/或有用，请给我们的 GitHub 仓库 点个星吧！您的支持对于 SeaStreamer 的持续开发至关重要。

以下是我们计划接下来开发的一些主要组件：

sea-streamer-file: 文件后端​

在 0.3 中发布了初步实现。但仍有一些待完成的任务：

  1. 可恢复性：当前尚未实现。一个潜在的实现方法是提交到本地 SQLite 数据库。
  2. 分片：目前只流式传输到分片零。
  3. 验证：一个用于验证和修复 SeaStreamer 二进制文件的工具程序。

sea-streamer-redis: Redis 集群​

在 sea-streamer 0.2 中发布了 Redis 支持！基本的流式分片已实现，但没有集群的分片并不太有用。

未来，我们希望支持 Redis 集群，目前这依然是个进行中的工作。这是一项相当困难的任务，因为客户端必须在与集群工作时承担责任。 在 Redis 中，分片和节点是动态的 M-N 映射 - 分片可以在 任何时候 在节点之间移动。 这使得测试更加困难。

在 Redis 中，同一组中的消费者共享相同的分片，即共享分片。未来，我们希望支持“拥有的分片”语义，其中每个消费者将尝试声明对分片的所有权，而组中的其他消费者不会介入。这模仿了 Kafka 的消费者组行为。

如果您希望提供帮助，请告诉我们！
