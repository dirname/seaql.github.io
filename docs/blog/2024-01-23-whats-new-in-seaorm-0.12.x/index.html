SeaORM 0.12.x 的新功能

January 23, 2024 · 9 min read
SeaQL 团队
Chris Tsang

自第一次发布 SeaORM 0.12 以来已经过去了一段时间。这篇博客总结了 SeaORM 0.12.2 到 0.12.12 版本引入的新特性和增强功能！

在 crates.io 上庆祝 200 万次下载 📦​

我们刚刚达成了在 crates.io 上累计 2,000,000 次下载的里程碑。这证明了 SeaORM 在专业使用中的广泛采用。感谢所有用户的信任，并成为我们社区的一部分。

新特性​

实体格式更新​

  • #1898 增加对根 JSON 数组的支持（需要 json-array / postgres-array 特性）！这涉及复杂的类型系统重构，以解决孤立规则的问题。
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "json_struct_vec")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    #[sea_orm(column_type = "Json")]
    pub struct_vec: Vec<JsonColumn>,
}

#[derive(Clone, Debug, PartialEq, Eq, Serialize, Deserialize, FromJsonQueryResult)]
pub struct JsonColumn {
    pub value: String,
}
  • #2009 为实体增加 comment 属性；create_table_from_entity 现在支持 MySQL 上的注释
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "applog", comment = "app logs")]
pub struct Model {
    #[sea_orm(primary_key, comment = "ID")]
    pub id: i32,
    #[sea_orm(comment = "action")]
    pub action: String,
    pub json: Json,
    pub created_at: DateTimeWithTimeZone,
}

游标分页器改进​

  • #2037 为游标增加降序排列：
// （默认行为）在 5 之前 ASC，即 id < 5

let mut cursor = Entity::find().cursor_by(Column::Id);
cursor.before(5);

assert_eq!(
    cursor.first(4).all(db).await?,
    [
        Model { id: 1 },
        Model { id: 2 },
        Model { id: 3 },
        Model { id: 4 },
    ]
);

// （新 API）在 5 之后 DESC，即 id < 5

let mut cursor = Entity::find().cursor_by(Column::Id);
cursor.after(5).desc();

assert_eq!(
    cursor.first(4).all(db).await?,
    [
        Model { id: 4 },
        Model { id: 3 },
        Model { id: 2 },
        Model { id: 1 },
    ]
);
  • #1826 为 SelectTwo 添加游标支持：
// 与关联关系连接；按第一个表的 id 游标

cake::Entity::find()
    .find_also_linked(entity_linked::CakeToFillingVendor)
    .cursor_by(cake::Column::Id)
    .before(10)
    .first(2)
    .all(&db)
    .await?

// 与关系连接；按第二个表的 id 游标

cake::Entity::find()
    .find_also_related(Fruit)
    .cursor_by_other(fruit::Column::Id)
    .before(10)
    .first(2)
    .all(&db)
    .await?

为数据库后端添加“代理”​

#1881, #2000 为数据库后端添加“代理”（需要特性标志 proxy）。

这使得在边缘/客户端使用 SeaORM 成为可能！请参见 GlueSQL 演示 获取示例。

增强功能​

  • #1954 [sea-orm-macro] 为 FromQueryResult 衍生宏添加 #[sea_orm(skip)]
#[derive(Clone, Debug, PartialEq, Eq, Deserialize, Serialize, FromQueryResult)]
pub struct PublicUser {
    pub id: i64,
    pub name: String,
    #[serde(skip_serializing_if = "Vec::is_empty")]
    #[sea_orm(skip)]
    pub something: Something,
}
  • #1598 [sea-orm-macro] 在 JsonValue 的 FromQueryResult 实现中添加对 Postgres 数组的支持
// 现有 API：

assert_eq!(
    Entity::find_by_id(1).one(db).await?,
    Some(Model {
        id: 1,
        name: "Collection 1".into(),
        integers: vec![1, 2, 3],
        teas: vec![Tea::BreakfastTea],
        colors: vec![Color::Black],
    })
);

// 新 API：

assert_eq!(
    Entity::find_by_id(1).into_json().one(db).await?,
    Some(json!({
        "id": 1,
        "name": "Collection 1",
        "integers": [1, 2, 3],
        "teas": ["BreakfastTea"],
        "colors": [0],
    }))
);
  • #1828 [sea-orm-migration] 检查索引是否存在
use sea_orm_migration::prelude::*;
#[derive(DeriveMigrationName)]
pub struct Migration;
#[async_trait::async_trait]
impl MigrationTrait for Migration {
    async fn up(&self, manager: &SchemaManager) -> Result<(), DbErr> {
        // ...

        // 确保索引未被创建
        assert!(!manager.has_index("cake", "cake_name_index").await?);

        manager
            .create_index(
                Index::create()
                    .name("cake_name_index")
                    .table(Cake::Table)
                    .col(Cake::Name)
                    .to_owned(),
            )
            .await?;

        Ok(())
    }

    async fn down(&self, manager: &SchemaManager) -> Result<(), DbErr> {
        // ...
    }
}

  • #1693 [sea-orm-cli] 支持生成复合外键的相关实体

发布计划​

在 SeaORM 0.12 的公告博客 中，我们表示希望减少重大版本的发布频率，同时保持功能更新和增强的步伐。我很高兴地说我们已经做到这一点！

下一个主要版本仍有一些计划中的破坏性更改。在经过一些 讨论 和考虑后，我们决定下一个主要版本将是 1.0 的候选版本！
