SeaORM 0.3.0 新特性

October 15, 2021 · 5 min read
SeaQL 团队
Chris Tsang

🎉 我们很高兴今天发布 SeaORM 0.3.0！以下是一些重要特性亮点 🌟：

事务​

[#222] 使用数据库事务执行原子操作

提供了两个事务 API：

  • closure 风格。成功时提交，失败时回滚。

    // <Fn, A, B> -> Result<A, B>
    db.transaction::<_, _, DbErr>(|txn| {
        Box::pin(async move {
            bakery::ActiveModel {
                name: Set("SeaSide Bakery".to_owned()),
                ..Default::default()
            }
            .save(txn)
            .await?;

            bakery::ActiveModel {
                name: Set("Top Bakery".to_owned()),
                ..Default::default()
            }
            .save(txn)
            .await?;

            Ok(())
        })
    })
    .await;

  • RAII 风格。begin 事务后跟 commit 或 rollback。如果 txn 超出范围，会自动回滚。

    let txn = db.begin().await?;

    // 使用 txn 执行某些操作

    txn.commit().await?;

流式处理​

[#222] 在任何 Select 上使用异步流以提高内存效率。

let mut stream = Fruit::find().stream(&db).await?;

while let Some(item) = stream.try_next().await? {
    let item: fruit::ActiveModel = item.into();
    // 使用 item 执行某些操作
}

保存和删除的自定义逻辑 API​

[#210] 我们重新定义了 ActiveModelBehavior 的特性方法。您现在可以在 insert、update、save、delete 操作之前和之后执行自定义验证。如果您在事务内部，即使操作已经完成，也可以中止操作。

impl ActiveModelBehavior for ActiveModel {
    // 重写默认值
    fn new() -> Self {
        Self {
            serial: Set(Uuid::new_v4()),
            ..ActiveModelTrait::default()
        }
    }

    // 在插入 / 更新之前触发
    fn before_save(self, insert: bool) -> Result<Self, DbErr> {
        if self.price.as_ref() <= &0.0 {
            Err(DbErr::Custom(format!(
                "[before_save] 无效的价格，插入: {}",
                insert
            )))
        } else {
            Ok(self)
        }
    }

    // 在插入 / 更新之后触发
    fn after_save(self, insert: bool) -> Result<Self, DbErr> {
        Ok(self)
    }

    // 在删除之前触发
    fn before_delete(self) -> Result<Self, DbErr> {
        Ok(self)
    }

    // 在删除之后触发
    fn after_delete(self) -> Result<Self, DbErr> {
        Ok(self)
    }
}

引入 DeriveIntoActiveModel 宏和 IntoActiveValue 特性​

[#240] 引入了一种新的派生宏 DeriveIntoActiveModel，用于在结构体上实现 IntoActiveModel。当您创建仅包含模型部分字段的自定义结构体时，这非常有用，例如在 REST API 中作为表单提交。

IntoActiveValue 特性允许将 Option<T> 转换为 ActiveValue<T>，方法为 into_active_value。

// 按照常规定义模型
#[derive(Clone, Debug, PartialEq, DeriveModel, DeriveActiveModel)]
#[sea_orm(table_name = "users")]
pub struct Model {
    pub id: Uuid,
    pub created_at: DateTimeWithTimeZone,
    pub updated_at: DateTimeWithTimeZone,
    pub email: String,
    pub password: String,
    pub full_name: Option<String>,
    pub phone: Option<String>,
}

// 创建一个省略某些字段的新结构体
#[derive(DeriveIntoActiveModel)]
pub struct NewUser {
    // id、created_at 和 updated_at 在该结构体中被省略，
    // 并将始终为 `ActiveValue::unset`
    pub email: String,
    pub password: String,
    // 全名通常是可选的，但在这里可以是必需的
    pub full_name: String,
    // Option 实现了 `IntoActiveValue`，在 `None` 时将为 `unset`
    pub phone: Option<String>,
}

#[derive(DeriveIntoActiveModel)]
pub struct UpdateUser {
    // Option<Option<T>> 允许 Some(None) 将列更新为 NULL
    pub phone: Option<Option<String>>,
}
