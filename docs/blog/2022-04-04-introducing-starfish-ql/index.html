介绍 StarfishQL

April 4, 2022 · 10 min read
SeaQL Team
Chris Tsang

我们很高兴今天向 Rust 社区介绍 StarfishQL。StarfishQL 是一个图数据库和查询引擎，用于在 web 上进行图分析和可视化。它是一个实验性项目，主要目的是探索在 crates.io 上发布的 Rust crates 的依赖关系网络。

动机​

StarfishQL 是一个提供图数据库和与之交互的图查询引擎的框架。

使用涉及 crates.io 上 crate 依赖关系的具体示例（Freeport）进行说明。通过这个例子，你可以看到 StarfishQL 的实际应用。

最终，我们关注的是执行图分析，也就是从普通的图数据中提取有意义的信息。为了实现这一点，我们认为可视化是一个至关重要的辅助工具。

StarfishQL 的查询引擎旨在能够通过灵活的查询语言来融入不同形式的可视化。然而，项目的开发主要集中在以下方面，正如我们在 demo apps 中展示的那样。

前 N 个依赖关系

从 N 个最连接的节点开始，正常方向遍历依赖图。

依赖关系与被依赖关系

从特定节点开始，在前向和反向方向遍历依赖树。

设计​

一般来说，查询引擎接收用特定查询语言（例如 SQL 语句）编写的输入查询，在数据库中执行必要的操作，然后将用户应用程序感兴趣的数据输出。你也可以将查询引擎视为一个抽象层，用户可以仅使用支持的查询语言设计查询，并让查询引擎处理其余部分。

在图查询引擎的情况下，输出数据是一个图（wiki）。

在 StarfishQL 的情况下，查询语言是我们在 JSON 格式中定义的自定义语言，这使得引擎具有高度的可访问性和可移植性。

实现​

在 Freeport 的示例中，StarfishQL 由以下三个组件组成。

图查询引擎​

作为 StarfishQL 的核心组件，图查询引擎是一个基于 Rocket web 框架和 SeaQL 生态系统 的 Rust 后端应用程序。

引擎在以下端点监听相应的操作：

  • /schema - 定义/重置模式
  • /mutate - 执行变异操作
  • /query - 执行查询

你也可以通过编程调用上述端点。

图数据存储在关系数据库中：

  • 元数据 - 每个实体和关系的定义，例如 crates 的属性和依赖关系
  • 节点数据 - 实体的一个实例，例如 crate 名称和版本号
  • 边数据 - 关系的一个实例，例如一个 crate 依赖于另一个 crate

crates.io 爬虫​

为了获得要插入数据库的 crate 数据，我们在 crates.io 的公共索引库的本地克隆上使用了一个 快速、无干扰的爬虫。

图可视化​

我们使用 d3.js 创建了力导向图以显示结果。上文中的两个 五彩斑斓的图表 就是这种产品。

衰减模式：无 则讲述了另一个有趣的故事：当连接性是 整个依赖树 时，我们正在关注那些深藏在许多 crates 之中的核心 crates。换句话说，这些是与最多 crates 一起构建的 crate。在这种情况下，与 Rust 更基本的低级方面交互的工具 crate 排名更高，例如 quote 具有关联语法树，proc-macro2 提供程序宏，以及 unicode-xid 进行 Unicode 检查。

没有依赖关系的 crates 数量

19,369 个 crates 中没有依赖于任何 crates，79,972 个 crates 总数中，占 24%。

没有被依赖关系的 crates 数量

53,910 个 crates 中没有其他 crates 依赖于它们，占 79,972 个 crates 总数的 67%。

我们想象许多这些 crates 是二进制/可执行文件，如果我们能找出检查方法就好了……🤔

截至 2022 年 3 月 30 日

结论​

StarfishQL 允许灵活和可移植的定义、操作、检索和可视化图数据。

用 Rust 构建的图查询引擎为任何 web 应用程序提供了一个良好的接口，以稳定的性能和内存安全访问关系图数据库中的数据。

可以肯定地说，StarfishQL 仍处于起步阶段，因此设计和实现中的每个细节都可能发生变化。幸运的是，像所有其他由优秀 Rust 开发者开发的开源项目一样，如果你也觉得这个概念有趣，你可以为它作出贡献。随着其加入 SeaQL 生态系统，我们更接近 Rust 数据工程的愿景。
