欢迎 2022 夏季编码贡献者

June 2, 2022 · 6 min read
SeaQL 团队
Chris Tsang

我们很高兴地宣布，今年夏天我们将有四位贡献者加入！其中两位由 Google 资助，另外两位由 SeaQL 资助。

基于 SeaORM 的 GraphQL 框架​

Panagiotis Karatakis

我是 Panagiotis，来自希腊雅典，目前攻读经济学的第二个学位。我的第一个学位是计算机科学，我对学习和实现企业软件解决方案充满热情。我学习 Rust 已经一年了，并在我和朋友为创业比赛创建的小型 Startup 项目中几乎每天都在使用它。

我将致力于创建一个 CLI 工具，用于探索数据库模式，然后生成一个可构建的 async-graphql API。该工具将允许与 SeaQL 和 Rust 生态系统以及 GraphQL 的快速集成。具体来说，我将使用 sea-schema 进行数据库探索，使用 sea-orm-codegen 进行实体生成，我的工作是将它们与 async-graphql 库结合在一起。你可以在 这里 了解更多。

用于模拟测试的 SQL 解释器​

Samyak Sarnayak

我是 Samyak Sarnayak，来自印度班加罗尔的计算机科学最后一年学生。我大约在 6-7 个月前开始学习 Rust，感觉找到了最适合我的编程语言 :D。它没有运行时，拥有出色的类型系统，错误信息非常友好，好用的工具，以及一些函数式编程模式和元编程。你可以在我的 GitHub 个人资料中找到更多关于我的信息。

我将在一个新的 SQL 解释器上工作，用于模拟测试。它将专门为测试构建，因此重点将放在正确性上——它可能速度较慢，但操作必须始终正确。我希望能够建立一个可工作的版本，并将其集成到 SeaORM 的现有测试中。关于这个项目的 讨论。

导师​

Chris Tsang

我坚信开源的重要性。我在 10 年前开始了我的 GitHub 之旅，发布了我的第一个编程库。我一直在寻找一种快速、符合人体工程学且富有表现力的编程语言。直到我找到 Rust。

看到 Rust 生态系统中对数据工程工具的需求，我在 2020 年创办了 SeaQL，并一直领导开发和维护这些库至今。


Billy Chan

你好，我是来自香港的 Billy。我从开始编程时就一直在使用开源库，但直到 2020 年，我才致力于成为一名 Rust 开源开发者。

我还是一名全栈开发者，专注于用户界面和数据库结构的需求规范制定、从零开始实施和测试前端和后端，最终发布 MVP，并维护它多年。

我喜欢与全球的 Rustaceans 一起工作，为 Rust 社区建立更好、更可持续的生态系统。如果你喜欢我们所做的事情，请考虑星标、评论、分享和贡献，这将不胜感激。


Sanford Pun

我是 Sanford，一名热情的软件工程师，喜欢解决问题！我已经在 Rust 上工作了几年。在我早期使用 Rust 的时候，我主要集中在图形/图像处理领域，我爱上了这门语言所提供的功能！今年，我正在 StarfishQL 项目中探索数据工程。

为 Rust 的无限潜力干杯！
