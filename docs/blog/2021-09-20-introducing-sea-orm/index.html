介绍 SeaORM 🐚

September 20, 2021 · 5 min read
Chris Tsang
SeaQL 团队

我们很高兴今天向 Rust 社区介绍 SeaORM 0.2.2。自 0.1.0 以来，我们很高兴收到了来自优秀的开发者对 SeaQuery 和 SeaORM 的反馈和贡献。

Rust 是一门可以用来构建任何东西的优秀语言。常见的问答之一是 “我们是否已经有 Web？” 如果说 Rocket（或您喜欢的任何 Web 框架）是 Rust 的 Rails，那么 SeaORM 就是 Rust 的 ActiveRecord。

SeaORM 是一个从零开始构建的异步 ORM，旨在与异步生态系统良好协作，无论是 actix、async-std、tokio 还是任何基于这些之上的 Web 框架。

让我们快速了解一下 SeaORM。

异步​

以下是如何并行执行多个查询：

// 并行执行多个查询
let cakes_and_fruits: (Vec<cake::Model>, Vec<fruit::Model>) =
    futures::try_join!(Cake::find().all(&db), Fruit::find().all(&db))?;

动态​

您可以使用 SeaQuery 构建复杂的查询而无需与 ORM 抵抗：

// 轻松构建子查询
let cakes_with_filling: Vec<cake::Model> = cake::Entity::find()
    .filter(
        Condition::any().add(
            cake::Column::Id.in_subquery(
                Query::select()
                    .column(cake_filling::Column::CakeId)
                    .from(cake_filling::Entity)
                    .to_owned(),
            ),
        ),
    )
    .all(&db)
    .await?;

可测试​

要编写单元测试，您可以使用我们的模拟接口：

// 设置模拟连接
let db = MockDatabase::new(DbBackend::Postgres)
    .append_query_results(vec![
        vec![
            cake::Model {
                id: 1,
                name: "New York Cheese".to_owned(),
            },
        ],
    ])
    .into_connection();

// 执行您的应用逻辑
assert_eq!(
    cake::Entity::find().one(&db).await?,
    Some(cake::Model {
        id: 1,
        name: "New York Cheese".to_owned(),
    })
);

// 与预期的事务日志进行比较
assert_eq!(
    db.into_transaction_log(),
    vec![
        Transaction::from_sql_and_values(
            DbBackend::Postgres,
            r#"SELECT "cake"."id", "cake"."name" FROM "cake" LIMIT $1"#,
            vec![1u64.into()]
        ),
    ]
);

面向服务​

这是一个带有分页的 Rocket 处理程序示例：

#[get("/?<page>&<posts_per_page>")]
async fn list(
    conn: Connection<Db>,
    page: Option<usize>,
    per_page: Option<usize>,
) -> Template {
    // 设置页面编号和每页项目
    let page = page.unwrap_or(1);
    let per_page = per_page.unwrap_or(10);

    // 设置分页器
    let paginator = Post::find()
        .order_by_asc(post::Column::Id)
        .paginate(&conn, per_page);
    let num_pages = paginator.num_pages().await.unwrap();

    // 获取分页的帖子
    let posts = paginator
        .fetch_page(page - 1)
        .await
        .expect("无法获取帖子");

    Template::render(
        "index",
        context! {
            page: page,
            per_page: per_page,
            posts: posts,
            num_pages: num_pages,
        },
    )
}
