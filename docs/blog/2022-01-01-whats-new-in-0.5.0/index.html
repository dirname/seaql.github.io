SeaORM 0.5.0 更新内容

January 1, 2022 · 6 min read
SeaQL Team
Chris Tsang

🎉 我们很高兴地宣布今天发布 SeaORM 0.5.0！以下是一些主要功能亮点 🌟：

插入和更新返回 Model​

[#339] 根据我们社区成员的要求，您现在可以在插入或更新操作后获取更新后的 Model。如果您想修改模型并将其保存回数据库，可以使用 into_active_model 方法将其转换为 ActiveModel。

破坏性更改：

  • ActiveModel::insert 和 ActiveModel::update 返回 Model 而不是 ActiveModel
  • 方法 ActiveModelBehavior::after_save 接受 Model 作为输入，而不是 ActiveModel
// 构建一个 `ActiveModel`
let active_model = ActiveModel {
    name: Set("经典香草蛋糕".to_owned()),
    ..Default::default()
};
// 执行插入
let cake: Model = active_model.insert(db).await?;
assert_eq!(
    cake,
    Model {
        id: 1,
        name: "经典香草蛋糕".to_owned(),
    }
);

// 将 `Model` 转换为 `ActiveModel`
let mut active_model = cake.into_active_model();
// 更改蛋糕的名称
active_model.name = Set("巧克力蛋糕".to_owned());
// 执行更新
let cake: Model = active_model.update(db).await?;
assert_eq!(
    cake,
    Model {
        id: 1,
        name: "巧克力蛋糕".to_owned(),
    }
);

// 执行删除
cake.delete(db).await?;

ActiveValue 完全重构​

[#340] ActiveValue 现在被定义为一个枚举而不是一个结构体。它的公共 API 保持不变，除了 Unset 被弃用，应该使用 ActiveValue::NotSet。

破坏性更改：

  • 将方法 sea_orm::unchanged_active_value_not_intended_for_public_use 重命名为 sea_orm::Unchanged
  • 将方法 ActiveValue::unset 重命名为 ActiveValue::not_set
  • 将方法 ActiveValue::is_unset 重命名为 ActiveValue::is_not_set
  • ActiveValue 的 PartialEq 将检查状态的相等性，而不仅仅是值的相等性
/// 定义 ActiveModel 中使用的有状态值。
pub enum ActiveValue<V>
where
    V: Into<Value>,
{
    /// 一个已定义的 [Value] 正在被激活设置
    Set(V),
    /// 一个已定义的 [Value] 保持不变
    Unchanged(V),
    /// 一个未定义的 [Value]
    NotSet,
}

SeaORM CLI & 代码生成更新​

安装最新版本的 sea-orm-cli：

cargo install sea-orm-cli

与实体文件生成相关的更新 (cargo generate entity)：

  • [#348] 发现并定义 PostgreSQL 枚举
  • [#386] 支持 SQLite 数据库，您可以从所有支持的数据库（包括 MySQL、PostgreSQL 和 SQLite）生成实体文件

        .with_max_level(tracing::Level::DEBUG)
