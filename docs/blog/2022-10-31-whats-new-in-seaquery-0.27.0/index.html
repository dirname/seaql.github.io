SeaQuery 0.27.0 的新特性

October 31, 2022 · 6 min read
SeaQL Team
Ivan Krivosheev

🎉 我们很高兴发布 SeaQuery 0.27.0！以下是一些功能亮点 🌟：

依赖升级​

[#356] 我们升级了一个主要依赖：

  • 升级 sqlx 到 0.6.1

您可能还需要在您的应用程序中升级相应的依赖。

驱动支持​

我们重新设计了 SeaQuery 中驱动的工作方式：在 0.27.0 之前，用户必须调用 sea_query_driver_* 宏。现在每个驱动 sqlx、postgres 和 rusqlite 有自己的支持库，与相应的库紧密集成。有关更多细节，请查看我们下面的集成示例。

[#383] 推弃 sea-query-driver，改用 sea-query-binder

[#422] Rusqlite 支持已迁移至 sea-query-rusqlite

[#433] Postgres 支持已迁移至 sea-query-postgres

// 之前
sea_query::sea_query_driver_postgres!();
use sea_query_driver_postgres::{bind_query, bind_query_as};

let (sql, values) = Query::select()
    .from(Character::Table)
    .expr(Func::count(Expr::col(Character::Id)))
    .build(PostgresQueryBuilder);

let row = bind_query(sqlx::query(&sql), &values)
    .fetch_one(&mut pool)
    .await
    .unwrap();

// 现在
use sea_query_binder::SqlxBinder;

let (sql, values) = Query::select()
    .from(Character::Table)
    .expr(Func::count(Expr::col(Character::Id)))
    .build_sqlx(PostgresQueryBuilder);

let row = sqlx::query_with(&sql, values)
    .fetch_one(&mut pool)
    .await
    .unwrap();

// 现在可以很好地使用 SQLx 的 `query_as_with`：
let rows = sqlx::query_as_with::<_, StructWithFromRow, _>(&sql, values)
    .fetch_all(&mut pool)
    .await
    .unwrap();

支持子查询运算符：EXISTS、ALL、ANY、SOME​

[#118] 新增子查询运算符：EXISTS、ALL、ANY、SOME

let query = Query::select()
    .column(Char::Id)
    .from(Char::Table)
    .and_where(
        Expr::col(Char::Id)
            .eq(
                Expr::any(
                    Query::select().column(Char::Id).from(Char::Table).take()
                )
            )
    )
    .to_owned();

assert_eq!(
    query.to_string(MysqlQueryBuilder),
    r#"SELECT `id` FROM `character` WHERE `id` = ANY(SELECT `id` FROM `character`)"#
);
assert_eq!(
    query.to_string(PostgresQueryBuilder),
    r#"SELECT "id" FROM "character" WHERE "id" = ANY(SELECT "id" FROM "character")"#
);

支持 ON CONFLICT WHERE​

[#366] 添加对 ON CONFLICT WHERE 的支持

let query = Query::insert()
    .into_table(Glyph::Table)
    .columns([Glyph::Aspect, Glyph::Image])
    .values_panic(vec![
        2.into(),
        3.into(),
    ])
    .on_conflict(
        OnConflict::column(Glyph::Id)
            .update_expr((Glyph::Image, Expr::val(1).add(2)))
            .target_and_where(Expr::tbl(Glyph::Table, Glyph::Aspect).is_null())
            .to_owned()
    )
    .to_owned();

assert_eq!(
    query.to_string(MysqlQueryBuilder),
    r#"INSERT INTO `glyph` (`aspect`, `image`) VALUES (2, 3) ON DUPLICATE KEY UPDATE `image` = 1 + 2"#
);
assert_eq!(
    query.to_string(PostgresQueryBuilder),
    r#"INSERT INTO "glyph" ("aspect", "image") VALUES (2, 3) ON CONFLICT ("id") WHERE "glyph"."aspect" IS NULL DO UPDATE SET "image" = 1 + 2"#
);
assert_eq!(
    query.to_string(SqliteQueryBuilder),
    r#"INSERT INTO "glyph" ("aspect", "image") VALUES (2, 3) ON CONFLICT ("id") WHERE "glyph"."aspect" IS NULL DO UPDATE SET "image" = 1 + 2"#
);

更改了 cond_where 链式语义​

[#414] 更改了 cond_where 链式语义

// 之前：将扩展当前条件
assert_eq!(
    Query::select()
        .cond_where(any![Expr::col(Glyph::Id).eq(1), Expr::col(Glyph::Id).eq(2)])
        .cond_where(Expr::col(Glyph::Id).eq(3))
        .to_owned()
        .to_string(PostgresQueryBuilder),
    r#"SELECT WHERE "id" = 1 OR "id" = 2 OR "id" = 3"#
);
// 之前：令人困惑，因为它依赖于调用顺序：
assert_eq!(
    Query::select()
        .cond_where(Expr::col(Glyph::Id).eq(3))
        .cond_where(any![Expr::col(Glyph::Id).eq(1), Expr::col(Glyph::Id).eq(2)])
        .to_owned()
        .to_string(PostgresQueryBuilder),
    r#"SELECT WHERE "id" = 3 AND ("id" = 1 OR "id" = 2)"#
);
// 现在：将始终与 `AND` 连接
assert_eq!(
    Query::select()
        .cond_where(Expr::col(Glyph::Id).eq(1))
        .cond_where(any![Expr::col(Glyph::Id).eq(2), Expr::col(Glyph::Id).eq(3)])
        .to_owned()
        .to_string(PostgresQueryBuilder),
    r#"SELECT WHERE "id" = 1 AND ("id" = 2 OR "id" = 3)"#
);
// 现在：它们现在是等效的
assert_eq!(
    Query::select()
        .cond_where(any![Expr::col(Glyph::Id).eq(2), Expr::col(Glyph::Id).eq(3)])
        .cond_where(Expr::col(Glyph::Id).eq(1))
        .to_owned()
        .to_string(PostgresQueryBuilder),
    r#"SELECT WHERE ("id" = 2 OR "id" = 3) AND "id" = 1"#
);

添加 OnConflict::value 和 OnConflict::values​

[#451] 为任何 Into<Value> 实现 From<T> 到 SimpleExpr

// 之前：注意元组
OnConflict::column(Glyph::Id).update_expr((Glyph::Image, Expr::val(1).add(2)))
// 现在：它接受 `Value` 以及 `SimpleExpr`
OnConflict::column(Glyph::Id).value(Glyph::Image, Expr::val(1).add(2))

改进 ColumnDef::default​

[#347] ColumnDef::default 现在接受 Into<SimpleExpr> 而不是 Into<Value>

// 现在我们可以写：
ColumnDef::new(Char::FontId)
    .timestamp()
    .default(Expr::current_timestamp())

破坏性更改​

  • [#386] 更改 in_tuples 接口以接受 IntoValueTuple
  • [#320] 移除了已弃用的方法
  • [#440] CURRENT_TIMESTAMP 从函数更改为关键字
  • [#375] 将 SQLite 的 boolean 类型从 integer 更改为 boolean
  • [#451] 弃用了 OnConflict::update_value、OnConflict::update_values、OnConflict::update_expr、OnConflict::update_exprs
  • [#451] 弃用了 InsertStatement::exprs、InsertStatement::exprs_panic
  • [#451] 弃用了 UpdateStatement::col_expr、UpdateStatement::value_expr、UpdateStatement::exprs
  • [#451] UpdateStatement::value 现在接受 Into<SimpleExpr> 而不是 Into<Value>
  • [#451] Expr::case、CaseStatement::case 和 CaseStatement::finally 现在接受 Into<SimpleExpr> 而不是 Into<Expr>
  • [#460] InsertStatement::values、UpdateStatement::values 现在接受 IntoIterator<Item = SimpleExpr> 而不是 IntoIterator<Item = Value>
  • [#409] 对于 SQLite 使用 SQLx 的原生 API 来处理时间
  • [#435] 将 ColumnType::Enum 的类型从 (String, Vec<String>) 更改为 Enum { name: DynIden, variants: Vec<DynIden>}

其他增强功能​

  • [#336] 为 SQLx 添加一维 Postgres 数组支持
  • [#373] 支持 CROSS JOIN
  • [#457] 为 SQLite 添加 DROP COLUMN 支持
  • [#466] 添加 YEAR、BIT 和 VARBIT 类型
  • [#338] 处理 Postgres 的模式名称用于模式语句
  • [#418] 添加 %、<< 和 >> 二进制运算符
  • [#329] 添加 RAND 函数
  • [#425] 为 Value 实现 Display
  • [#427] 向 UnionType 添加 INTERSECT 和 EXCEPT
  • [#448] OrderedStatement::order_by_customs、OrderedStatement::order_by_columns、OverStatement::partition_by_customs、OverStatement::partition_by_columns 现在接受 IntoIterator<Item = T> 而不是 Vec<T>
  • [#452] TableAlterStatement::rename_column、TableAlterStatement::drop_column、ColumnDef::new、ColumnDef::new_with_type 现在接受 IntoIden 而不是 Iden
  • [#426] 清理 IndexBuilder 特性方法
  • [#436] 引入 SqlWriter 特性
  • [#448] 从示例中移除不需要的 vec!

错误修复​

  • [#449] distinct_on 正确处理 ColumnRef
  • [#461] 为 SQLite 移除了 DROP INDEX 的 ON
  • [#468] 更改日期时间字符串格式以包含微秒
  • [#452] 对 PostgreSQL 进行的 ALTER TABLE 有 UNIQUE 约束
