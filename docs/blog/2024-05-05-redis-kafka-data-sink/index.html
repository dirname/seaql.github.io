构建 Redis / Kafka 数据接收器

May 5, 2024 · 8 min read
Chris Tsang
SeaQL团队

本教程将向您展示如何使用 Rust 构建一个系统，该系统实现以下功能：

  1. 订阅实时 websocket 数据流
  2. 将数据流传输到 Kafka / Redis
  3. 将数据保存到 SQL 数据库

在这里，我们将采用微服务架构，并将功能拆分为两个应用：

┌─────────────────────┐      ─────────────────      ┌───────────────┐
│ Websocket 数据流    │ --->   Redis / Kafka   ---> │ SQL 数据接收器 │
└─────────────────────┘      ─────────────────      └───────────────┘

在流处理过程中，我们通常使用“源”和“接收器”这两个术语，但数据接收器只是将数据持久化到存储中的流消费者。

在源端，我们将使用 SeaStreamer。在接收器端，我们将使用 SeaORM。以下是支持的技术；在本文的其余部分，我们将使用 Redis 和 SQLite，因为它们易于设置。

SeaStreamerSeaORM
Kafka, RedisMySQL, Postgres, SQLite, SQL Server1

要开始，您可以通过 Docker 快速启动一个 Redis 实例：

docker run -d --rm --name redis -p 6379:6379 redis

1. Websocket 订阅​

让我们在 Rust 中编写一个 websocket 订阅者。在这里，我们将使用出色的 async-tungstenite 库。

我们将订阅 Kraken 的 GBP/USD 汇率数据流，API 文档可以在 这里 找到。注意：这些数据并不是真实的外汇数据，但应该足够用于演示。

步骤 1，创建一个 websocket 连接：

let (mut ws, _) = async_tungstenite::tokio::connect_async("wss://ws.kraken.com/").await?;

步骤 2，发送订阅请求：

ws.send(Message::Text(
    r#"{ "event": "subscribe", "pair": ["GBP/USD"], "subscription": { "name": "spread" } }"#.to_owned(),
)).await?;

步骤 3，流处理消息：

loop {
    match ws.next().await {
        Some(Ok(Message::Text(data))) => {
            if data == r#"{"event":"heartbeat"}"# {
                continue; // 跳过心跳事件
            }
            println!("{data}");
        }
        Some(Err(e)) => bail!("Socket error: {e}"),
        None => bail!("Stream ended"),
        e => bail!("Unexpected message {e:?}"),
    }
}

2. Redis / Kafka 流生产者​

步骤 1，创建一个连接到 Redis / Kafka 的 SeaStreamer 实例：

let streamer = SeaStreamer::connect(
        "redis://localhost", SeaConnectOptions::default()
    ).await?;

Redis 和 Kafka 各自有很多不同的选项，您可以参考 SeaStreamer 的 文档。

步骤 2，创建生产者：

let producer: SeaProducer = streamer
    .create_producer(
        "GBP_USD".parse()?, // 流键
        Default::default(),  // 生产者选项
    )
    .await?;

生产者没有特定的选项。

步骤 3，解码消息：

let spread: SpreadMessage = serde_json::from_str(&data)?;
let message = serde_json::to_string(&spread)?;

在这里，我们使用出色的 serde 库来进行消息解析和转换：

// 原始消息格式为: [80478222,["1.25475","1.25489","1714946803.030088","949.74917071","223.36195920"],"spread","GBP/USD"]

#[derive(Debug, Serialize, Deserialize)]
struct SpreadMessage {
    #[allow(dead_code)]
    #[serde(skip_serializing)]
    channel_id: u32, // 占位符; 不需要
    spread: Spread, // 嵌套对象
    channel_name: String,
    pair: String,
}

#[derive(Debug, Serialize, Deserialize)]
struct Spread {
    bid: Decimal,
    ask: Decimal,
    #[serde(with = "timestamp_serde")] // 自定义序列化
    timestamp: Timestamp,
    bid_vol: Decimal,
    ask_vol: Decimal,
}

步骤 4，发送消息：

loop {
    match ws.next().await {
        Some(Ok(Message::Text(data))) => {
            let spread: SpreadMessage = serde_json::from_str(&data)?;
            let message = serde_json::to_string(&spread)?;
            producer.send(message)?; // <--
        }
    }
}

请注意，producer.send 调用不是 async/await，这是一个关键细节！这消除了流处理瓶颈。在幕后，消息将被缓冲并在不同的线程中处理，这样您的输入流可以尽可能接近实时运行。

以下是完整的 price-feed 应用，您可以从 SeaStreamer 仓库中查看：

$ cd examples/price-feed
$ cargo run

Connecting ..
Connected.
Subscribed.
{"spread":{"bid":"1.25495","ask":"1.25513","timestamp":"2024-05-05T16:31:00.961214","bid_vol":"61.50588918","ask_vol":"787.90883861"},"channel_name":"spread","pair":"GBP/USD"}

3. SQL 数据接收器​

步骤 1，创建一个流消费者：

let streamer = SeaStreamer::connect(streamer_uri, Default::default()).await?;

let consumer = streamer
    .create_consumer(&[stream_key], SeaConsumerOptions::default())
    .await?;

Redis 和 Kafka 各自有很多不同的选项，您可以参考 SeaStreamer 的 示例。在这里我们使用默认选项，即一个实时无状态流消费者。

步骤 2，创建数据库：

let mut opt = ConnectOptions::new("sqlite://my_db.sqlite?mode=rwc"));
opt.max_connections(1).sqlx_logging(false);
let db = Database::connect(opt).await?;

我们将 max_connections 设置为 1，因为我们的数据接收器不需要并发插入。

以下是 Entity：

#[derive(Debug, Clone, PartialEq, Eq, DeriveEntityModel, Deserialize)]
#[sea_orm(table_name = "event")]
pub struct Model {
    #[sea_orm(primary_key)]
    #[serde(default)]
    pub id: i32,
    pub timestamp: String,
    pub bid: String,
    pub ask: String,
    pub bid_vol: String,
    pub ask_vol: String,
}

该表将命名为 event，并且我们在模型上派生 Deserialize。

我们将使用以下辅助方法创建数据库表，其中架构是从实体派生的：

async fn create_tables(db: &DbConn) -> Result<(), DbErr> {
    let builder = db.get_database_backend();
    let schema = Schema::new(builder);

    let stmt = builder.build(
        schema.create_table_from_entity(Entity).if_not_exists(),
    );
    log::info!("{stmt}");
    db.execute(stmt).await?;

    Ok(())
}

这对于 SQLite 特别方便，因为应用程序拥有数据库架构。对于其他数据库，您可能会使用 SeaORM 迁移系统。

步骤 3，将数据插入数据库：

loop {
    let message = consumer.next().await?;
    let payload = message.message();
    let json = payload.as_str()?;
    let item: Item = serde_json::from_str(json)?;
    let mut spread = item.spread.into_active_model();
    spread.id = NotSet; // 让数据库分配主键
    spread.save(&db).await?;
}

在几行代码中，我们：

  1. 从 Redis 接收消息
  2. 将消息解码为 JSON
  3. 将消息转换为 SeaORM 模型
  4. 将模型插入数据库

结论​

在本文中，我们涵盖了：

  1. 流处理中的微服务架构
  2. Rust 中的异步实时编程
  3. SeaQL 和 Rust 生态系统的强大2

以下是一些建议，您可以在此基础上进一步拓展：

  1. 将数据流向 MySQL 或 Postgres 等“大型数据库”
  2. 订阅更多流并接收至更多表
  3. 缓冲事件并批量插入数据以提高吞吐量，进一步阅读：
