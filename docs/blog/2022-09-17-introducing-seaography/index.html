介绍 Seaography 🧭

September 17, 2022 · 6 min read
SeaQL 团队
Chris Tsang

多么丰硕的编程夏季！今天，我们很高兴向 SeaQL 社区介绍 Seaography。Seaography 是一个用于构建 GraphQL 解析器的 GraphQL 框架，使用 SeaORM。它配备了一种 CLI 工具，可以根据现有的 MySQL、Postgres 和 SQLite 数据库生成即刻可编译的 Rust 项目。

动机​

我们观察到其他生态系统有类似的工具，如 PostGraphile 和 Hasura，允许用户通过 GraphQL 轻松查询数据库。我们决定将这种无缝体验带入 Rust 生态系统。

对于现有的 SeaORM 用户而言，添加 GraphQL API 是非常简单的。首先，在您的 crate 中添加 seaography 和 async-graphql 依赖项。然后，将一些额外的衍生宏添加到 SeaORM 实体中。最后，启动一个 GraphQL 服务器来处理查询！

如果您是 SeaORM 的新手，没问题，我们会支持您。您只需提供数据库连接，然后 seaography-cli 将根据提供的数据库生成 SeaORM 实体以及完整的 Rust 项目！

设计​

在我们的 初步讨论中，我们考虑了两种方法：1）黑箱查询引擎 2）代码生成器。黑箱查询引擎的缺点是很难自定义或扩展其行为，从而使长期开发和操作变得困难。我们选择了代码生成器的方法，让用户完全控制，并与多样化的异步 Rust 生态系统创造无尽的可能性。

该项目分为以下 crates：

  • seaography：外观 crate；导出宏、结构和辅助函数，将 SeaORM 实体转换为 GraphQL 节点。

  • seaography-cli：CLI 工具；根据用户提供的数据库生成 SeaORM 实体以及完整 Rust 项目。

  • seaography-discoverer：CLI 工具使用的辅助 crate，用于发现数据库模式并转换为通用格式。

  • seaography-generator：CLI 工具使用的辅助 crate，用于消费数据库模式并生成完整的 Rust 项目。

  • seaography-derive：一组过程宏，用于在 SeaORM 实体上派生类型和特征实现，将它们转换为 GraphQL 节点。

特性​

  • 关系查询（1 对 1，1 对 N）
  • 查询根实体的分页
  • 使用运算符过滤（例如 gt、lt、eq）
  • 按任意列排序

开始使用​

要快速开始，我们为您准备了以下示例，以及初始化数据库的 SQL 脚本。

  • MySQL
  • PostgreSQL
  • SQLite

所有示例在运行时都提供了基于 web 的 GraphQL 游乐场，您可以在其中检查 GraphQL 模式并进行查询。如果您迫不及待想要玩，可以访问我们托管的 演示 GraphQL 游乐场。

有关更多文档，请访问 www.sea-ql.org/Seaography。

下一步？​

该项目通过了第一个里程碑，实现了基本功能，但仍然有很长的路要走。下一个里程碑将包括：

  • 查询增强
    • 过滤相关查询
    • 基于相关查询属性的过滤
    • 分页相关查询
    • 按相关查询排序
  • 基于游标的分页
  • 单实体查询
  • 变更操作
    • 插入单个实体
    • 批量插入实体
    • 更新单个实体
    • 使用过滤器更新批量实体
    • 删除单个实体
    • 批量删除实体

结论​

Seaography 是一个将 SeaORM 实体转换为 GraphQL 节点的高效库。它提供了一套实用工具，并结合代码生成器，使 GraphQL API 的构建变得轻而易举。

然而，Seaography 仍然是一个新生项目。像所有由热情的 Rust 开发者开发的开源项目一样，如果您也对这个概念感兴趣，欢迎您为其贡献。随着它加入 SeaQL 生态系统，我们离 Rust 成为数据工程最佳工具的愿景又近了一步。
