🎉 我们很高兴今天发布 SeaORM [`0.9.0`](https://github.com/SeaQL/sea-orm/releases/tag/0.9.0)！以下是一些功能亮点 🌟：

## 依赖升级

[[#834](https://github.com/SeaQL/sea-orm/pull/834)] 我们升级了一些主要的依赖：
- 将 [`sqlx`](https://github.com/launchbadge/sqlx) 升级到 0.6
- 将 [`time`](https://github.com/time-rs/time) 升级到 0.3
- 将 [`uuid`](https://github.com/uuid-rs/uuid) 升级到 1.0
- 将 [`sea-query`](https://github.com/SeaQL/sea-query) 升级到 0.26
- 将 [`sea-schema`](https://github.com/SeaQL/sea-schema) 升级到 0.9

请注意，您可能还需要升级应用程序中相应的依赖。

## 游标分页

[[#822](https://github.com/SeaQL/sea-orm/pull/822)] 基于主键等列对模型进行分页。

```rust
// 创建一个按 `cake`.`id` 排序的游标
let mut cursor = cake::Entity::find().cursor_by(cake::Column::Id);

// 按 `cake`.`id` > 1 且 `cake`.`id` < 100 过滤分页结果
cursor.after(1).before(100);

// 获取前 10 行（按 `cake`.`id` 升序）
let rows: Vec<cake::Model> = cursor.first(10).all(db).await?;

// 获取后 10 行（按 `cake`.`id` 降序但行以升序返回）
let rows: Vec<cake::Model> = cursor.last(10).all(db).await?;

冲突时插入​

[#791] 使用冲突行为插入活动模型。

let orange = cake::ActiveModel {
    id: ActiveValue::set(2),
    name: ActiveValue::set("Orange".to_owned()),
};

// 在冲突时不执行任何操作： 
//   - INSERT INTO "cake" ("id", "name") VALUES (2, 'Orange') ON CONFLICT ("name") DO NOTHING
cake::Entity::insert(orange.clone())
    .on_conflict(
        sea_query::OnConflict::column(cake::Column::Name)
            .do_nothing()
            .to_owned()
    )
    .exec(db)
    .await?;

// 在冲突时进行更新：
//   - INSERT INTO "cake" ("id", "name") VALUES (2, 'Orange') ON CONFLICT ("name") DO UPDATE SET "name" = "excluded"."name"
cake::Entity::insert(orange)
    .on_conflict(
        sea_query::OnConflict::column(cake::Column::Name)
            .update_column(cake::Column::Name)
            .to_owned()
    )
    .exec(db)
    .await?;

使用自定义条件和表别名的连接表​

[#793, #852] 点击 自定义连接条件 和 自定义连接 了解更多。

assert_eq!(
    cake::Entity::find()
        .column_as(
            Expr::tbl(Alias::new("fruit_alias"), fruit::Column::Name).into_simple_expr(),
            "fruit_name"
        )
        .join_as(
            JoinType::LeftJoin,
            cake::Relation::Fruit
                .def()
                .on_condition(|_left, right| {
                    Expr::tbl(right, fruit::Column::Name)
                        .like("%tropical%")
                        .into_condition()
                }),
            Alias::new("fruit_alias")
        )
        .build(DbBackend::MySql)
        .to_string(),
    [
        "SELECT `cake`.`id`, `cake`.`name`, `fruit_alias`.`name` AS `fruit_name` FROM `cake`",
        "LEFT JOIN `fruit` AS `fruit_alias` ON `cake`.`id` = `fruit_alias`.`cake_id` AND `fruit_alias`.`name` LIKE '%tropical%'",
    ]
    .join(" ")
);

(反)序列化自定义 JSON 类型​

[#794] 存储在数据库中的 JSON 可以反序列化为 Rust 中的自定义结构体。

#[derive(Clone, Debug, PartialEq, DeriveEntityModel)]
#[sea_orm(table_name = "json_struct")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    // 在 `serde_json::Value` 中定义的 JSON 列
    pub json: Json,
    // 在自定义结构体中定义的 JSON 列
    pub json_value: KeyValue,
    pub json_value_opt: Option<KeyValue>,
}

// 自定义结构体必须派生 `FromJsonQueryResult`、`Serialize` 和 `Deserialize`
#[derive(Clone, Debug, PartialEq, Serialize, Deserialize, FromJsonQueryResult)]
pub struct KeyValue {
    pub id: i32,
    pub name: String,
    pub price: f32,
    pub notes: Option<String>,
}

派生迁移名称​

[#736] 引入 DeriveMigrationName 过程宏，从文件名推断迁移名称。

use sea_orm_migration::prelude::*;

// 以前是...
pub struct Migration;

impl MigrationName for Migration {
    fn name(&self) -> &str {
        "m20220120_000001_create_post_table"
    }
}

// 现在... 派生 `DeriveMigrationName`，
// 不再需要显式指定迁移名称
#[derive(DeriveMigrationName)]
pub struct Migration;

#[async_trait::async_trait]
impl MigrationTrait for Migration {
    async fn up(&self, manager: &SchemaManager) -> Result<(), DbErr> {
        manager
            .create_table( ... )
            .await
    }

    async fn down(&self, manager: &SchemaManager) -> Result<(), DbErr> {
        manager
            .drop_table( ... )
            .await
    }
}
