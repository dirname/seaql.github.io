在本教程中，我们将使用 [Seaography](https://github.com/SeaQL/seaography) 为我们的 Loco 启动应用程序添加一个 GraphQL 端点。如果你还没有阅读该系列的第一教程 [Loco & SeaORM 入门](https://www.sea-ql.org/blog/2024-05-28-getting-started-with-loco-seaorm/)，请先阅读。

完整的源代码可以在 [这里](https://github.com/SeaQL/sea-orm/tree/master/examples/loco_seaography) 找到。

## 什么是 Seaography

Seaography 是一个 GraphQL 框架，可以使用 SeaORM 实体构建 GraphQL 解析器。它附带一个 CLI 工具，可以从现有的 MySQL、Postgres 和 SQLite 数据库生成可编译的 Rust GraphQL 服务器。

## 添加依赖

修改 `Cargo.toml` 并添加几个依赖项：`seaography`、`async-graphql`、`async-graphql-axum` 和 `lazy_static`。

```toml title="loco_seaography/Cargo.toml"
seaography = { version = "1.0.0-rc.4", features = ["with-decimal", "with-chrono"] }
async-graphql = { version = "7.0", features = ["decimal", "chrono", "dataloader", "dynamic-schema"] }
async-graphql-axum = { version = "7.0" }
lazy_static = { version = "1.4" }
tower-service = { version = "0.3" }

为 Seaography 设置 SeaORM 实体​

Seaography 实体基本上是带有一些附加项的 SeaORM 实体。它们与 SeaORM 完全兼容。

你可以使用 sea-orm-cli 并带上额外的 --seaography 标志来生成 Seaography 实体。

sea-orm-cli generate entity -o src/models/_entities -u postgres://loco:loco@localhost:5432/loco_seaography_development --seaography
loco_seaography/src/models/_entities/notes.rs
use sea_orm::entity::prelude::*;
use serde::{Serialize, Deserialize};

#[derive(Clone, Debug, PartialEq, DeriveEntityModel, Eq, Serialize, Deserialize)]
#[sea_orm(table_name = "notes")]
pub struct Model {
    pub created_at: DateTime,
    pub updated_at: DateTime,
    #[sea_orm(primary_key)]
    pub id: i32,
    pub title: Option<String>,
    pub content: Option<String>,
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {
    #[sea_orm(has_many = "super::files::Entity")]
    Files,
}

impl Related<super::files::Entity> for Entity {
    fn to() -> RelationDef {
        Relation::Files.def()
    }
}

+ // 定义 `RelatedEntity` 以将一个实体与另一个实体关联
+ #[derive(Copy, Clone, Debug, EnumIter, DeriveRelatedEntity)]
+ pub enum RelatedEntity {
+     #[sea_orm(entity = "super::files::Entity")]
+     Files,
+ }

我们可以看到在实体文件中生成了一个新的枚举 RelatedEntity。这帮助 Seaography 找到相关实体以进行关系查询。

实现 GraphQL 查询根​

我们已经完成了 SeaORM 实体的设置，现在我们来实现 Seaography 的查询根，在这里我们将 SeaORM 和 Async GraphQL 进行连接，并借助 Seaography 来实现。

loco_seaography/src/graphql/query_root.rs
use async_graphql::dynamic::*;
use sea_orm::DatabaseConnection;
use seaography::{Builder, BuilderContext};

use crate::models::_entities::*;

lazy_static::lazy_static! { static ref CONTEXT: BuilderContext = BuilderContext::default(); }

pub fn schema(
    database: DatabaseConnection,
    depth: usize,
    complexity: usize,
) -> Result<Schema, SchemaError> {
    // Seaography 查询根的构建器
    let mut builder = Builder::new(&CONTEXT, database.clone());
    // 注册 SeaORM 实体
    seaography::register_entities!(
        builder,
        // 在此处列出我们想要包含在 GraphQL 端点中的所有模型
        [files, notes, users]
    );
    // 配置 async GraphQL 限制
    let schema = builder
        .schema_builder()
        // 深度是字段的嵌套级别数量
        .limit_depth(depth)
        // 复杂度是查询中的字段数量
        .limit_complexity(complexity);
    // 最后包括 SeaORM 数据库连接
    schema.data(database).finish()
}

编写 Loco 控制器以处理 GraphQL 端点​

为了方便，我们使用 async-graphql 中内置的 GraphQL playground UI 来测试 GraphQL 端点。并使用 async_graphql_axum 和 Seaography 来处理 GraphQL 请求。

loco_seaography/src/controllers/graphql.rs
use async_graphql::http::{playground_source, GraphQLPlaygroundConfig};
use axum::{body::Body, extract::Request};
use loco_rs::prelude::*;
use tower_service::Service;

use crate::graphql::query_root;

// GraphQL playground UI
async fn graphql_playground() -> Result<Response> {
    // `GraphQLPlaygroundConfig` 接受一个参数
    // 即 GraphQL 处理程序的 URL: `/api/graphql`
    let res = playground_source(GraphQLPlaygroundConfig::new("/api/graphql"));

    Ok(Response::new(res.into()))
}

async fn graphql_handler(
    State(ctx): State<AppContext>,
    req: Request<Body>,
) -> Result<Response> {
    const DEPTH: usize = 10;
    const COMPLEXITY: usize = 100;
    // 构建 GraphQL 查询根
    let schema = query_root::schema(ctx.db.clone(), DEPTH, COMPLEXITY).unwrap();
    // GraphQL 处理程序
    let mut graphql_handler = async_graphql_axum::GraphQL::new(schema);
    // 执行 GraphQL 请求并获取结果
    let res = graphql_handler.call(req).await.unwrap();

    Ok(res)
}

pub fn routes() -> Routes {
    // 定义路由
    Routes::new()
        // 我们将所有 GraphQL 路由放在 `graphql` 前缀后面
        .prefix("graphql")
        // GraphQL playground 页面是 GET 请求
        .add("/", get(graphql_playground))
        // GraphQL 处理程序是 POST 请求
        .add("/", post(graphql_handler))
}

创建笔记​

使用 GraphQL 变异创建新的笔记。

mutation {
  notesCreateOne(
    data: {
      id: 1
      title: "Notes 001"
      content: "Content 001"
      createdAt: "2024-06-24 00:00:00"
      updatedAt: "2024-06-24 00:00:00"
    }
  ) {
    id
    title
    content
    createdAt
    updatedAt
  }
}

查询笔记​

查询包含相关文件的笔记。

query {
  notes {
    nodes {
      id
      title
      content
      files {
        nodes {
          id
          filePath
        }
      }
    }
  }
}

向 GraphQL 端点添加用户认证​

我们的 GraphQL 处理程序可以在没有用户认证的情况下访问。接下来，我们希望只允许已登录用户访问 GraphQL 处理程序。

为此，我们将 _auth: auth::JWT 添加到 graphql_handler 函数中。

loco_seaography/src/controllers/graphql.rs
async fn graphql_handler(
+   _auth: auth::JWT,
    State(ctx): State<AppContext>,
    req: Request<Body>,
) -> Result<Response> {
    const DEPTH: usize = 10;
    const COMPLEXITY: usize = 100;
    // 构建 GraphQL 查询根
    let schema = query_root::schema(ctx.db.clone(), DEPTH, COMPLEXITY).unwrap();
    // GraphQL 处理程序
    let mut graphql_handler = async_graphql_axum::GraphQL::new(schema);
    // 执行 GraphQL 请求并获取结果
    let res = graphql_handler.call(req).await.unwrap();

    Ok(res)
}

然后，运行 Loco 应用程序并再次访问 GraphQL playground。你应该会看到未授权错误。

结论​

在 Loco 应用程序中添加 GraphQL 支持非常简单，得益于 Seaography。这是一个人性化的库，将 SeaORM 实体转换为 GraphQL 节点，并提供一系列实用程序，结合代码生成器，使 GraphQL API 的构建变得轻而易举。

SQL Server 支持​

SeaORM 的 SQL Server 现已作为闭测试版提供。如果你感兴趣，请在 这里 注册。

从 sea-orm 迁移到 sea-orm-x 十分简单，只需两步。首先，将现有的 sea-orm 依赖项更新为 sea-orm-x 并启用 sqlz-mssql 特性。请注意，你可能需要为上游依赖项打补丁 SeaORM 依赖项。

Cargo.toml
sea-orm = { path = "<SEA_ORM_X_ROOT>/sea-orm-x", features = ["runtime-async-std-rustls", "sqlz-mssql"] }
sea-orm-migration = { path = "<SEA_ORM_X_ROOT>/sea-orm-x/sea-orm-migration" }

# 为上游依赖项补丁 SeaORM 依赖项
[patch.crates-io]
sea-orm = { path = "<SEA_ORM_X_ROOT>/sea-orm-x" }
sea-orm-migration = { path = "<SEA_ORM_X_ROOT>/sea-orm-x/sea-orm-migration" }

其次，更新连接字符串以连接到 MSSQL 数据库。

# 如果模式是 `dbo`，只需写：
mssql://username:password@host/database

# 或者，通过提供额外的 `currentSchema` 查询参数指定模式名称。
mssql://username:password@host/database?currentSchema=my_schema

# 你可以通过提供额外的 trustCertificate 查询参数来信任对等证书。
mssql://username:password@host/database?trustCertificate=true
