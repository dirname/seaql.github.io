完整的源代码可以在 [这里](https://github.com/SeaQL/sea-orm/tree/master/examples/loco_starter) 找到。REST API 的文档可以在 [这里](https://documenter.getpostman.com/view/34752358/2sA3QmEF5q) 查看。

## 什么是 Loco？

[Loco](https://loco.rs/) 是一个受 Rails 启发的 Rust 网络框架。它包含许多 Rails 的特性和 Rust 的易用性。Loco 与 SeaORM 无缝集成，提供一流的开发体验！

- 通过 [axum](https://github.com/tokio-rs/axum) 进行控制器和路由
- 通过 [SeaORM](https://www.sea-ql.org/SeaORM/) 进行模型、迁移和 ActiveRecord
- 通过 [serde](https://serde.rs/json.html) 进行视图
- 无缝的后台作业，使用 [sidekiq-rs](https://github.com/film42/sidekiq-rs)，多模式：进程内、进程外、通过 Tokio 的异步
- [...更多](https://loco.rs/blog/hello-world/)

用户管理​

启动模板附带了一个基本的用户管理模块。

注册​

常见做法是向提供的电子邮件发送验证邮件。然而，这需要一个 SMTP 服务器，而这不是本博客文章的重点。因此，我将跳过邮件验证：

loco_starter/src/controllers/auth.rs
#[debug_handler]
async fn register(
    State(ctx): State<AppContext>,
    Json(params): Json<RegisterParams>,
) -> Result<Response> {
    let res = users::Model::create_with_password(&ctx.db, &params).await;

    let user = match res {
        Ok(user) => user,
        Err(err) => {
            tracing::info!(
                message = err.to_string(),
                user_email = &params.email,
                "无法注册用户",
            );
            return format::json(());
        }
    };

+   // 跳过电子邮件验证，所有新注册的用户都被视为已验证
+   let _user = user
+       .into_active_model()
+       .verified(&ctx.db)
+       .await?;

+   // 跳过发送验证邮件，因为我们没有邮件服务器
+   /*
    let user = user
        .into_active_model()
        .set_email_verification_sent(&ctx.db)
        .await?;

    AuthMailer::send_welcome(&ctx, &user).await?;
+   */

    format::json(())
}

文件表迁移​

为新的 files 表创建一个迁移文件。每行 files 参考数据库中的特定 notes。

loco_starter/migration/src/m20240520_173001_files.rs
use sea_orm_migration::{prelude::*, schema::*};

use super::m20231103_114510_notes::Notes;

#[derive(DeriveMigrationName)]
pub struct Migration;

#[async_trait::async_trait]
impl MigrationTrait for Migration {
    async fn up(&self, manager: &SchemaManager) -> Result<(), DbErr> {
        manager
            .create_table(
                table_auto(Files::Table)
                    .col(pk_auto(Files::Id))
                    .col(integer(Files::NotesId))
                    .col(string(Files::FilePath))
                    .foreign_key(
                        ForeignKey::create()
                            .name("FK_files_notes_id")
                            .from(Files::Table, Files::NotesId)
                            .to(Notes::Table, Notes::Id),
                    )
                    .to_owned(),
            )
            .await
    }

    async fn down(&self, manager: &SchemaManager) -> Result<(), DbErr> {
        manager
            .drop_table(Table::drop().table(Files::Table).to_owned())
            .await
    }
}

#[derive(DeriveIden)]
pub enum Files {
    Table,
    Id,
    NotesId,
    FilePath,
}

文件模型定义​

定义 files 实体模型。

loco_starter/src/models/_entities/files.rs
use sea_orm::entity::prelude::*;
use serde::{Deserialize, Serialize};

#[derive(Clone, Debug, PartialEq, DeriveEntityModel, Eq, Serialize, Deserialize)]
#[sea_orm(table_name = "files")]
pub struct Model {
    pub created_at: DateTime,
    pub updated_at: DateTime,
    #[sea_orm(primary_key)]
    pub id: i32,
    pub notes_id: i32,
    pub file_path: String,
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {
    #[sea_orm(
        belongs_to = "super::notes::Entity",
        from = "Column::NotesId",
        to = "super::notes::Column::Id"
    )]
    Notes,
}

impl Related<super::notes::Entity> for Entity {
    fn to() -> RelationDef {
        Relation::Notes.def()
    }
}

文件控制器​

控制器是我们处理文件上传、列出和查看的地方。

上传文件​

以下上传处理程序允许在单个 POST 请求中上传多个文件。

loco_starter/src/controllers/files.rs
#[debug_handler]
pub async fn upload(
    _auth: auth::JWT,
    Path(notes_id): Path<i32>,
    State(ctx): State<AppContext>,
    mut multipart: Multipart,
) -> Result<Response> {
    // 收集所有上传的文件
    let mut files = Vec::new();

    // 迭代 POST 主体中的所有文件
    while let Some(field) = multipart.next_field().await.map_err(|err| {
        tracing::error!(error = ?err,"无法读取 multipart");
        Error::BadRequest("无法读取 multipart".into())
    })? {
        // 获取文件名
        let file_name = match field.file_name() {
            Some(file_name) => file_name.to_string(),
            _ => return Err(Error::BadRequest("未找到文件名".into())),
        };

        // 获取文件内容为字节
        let content = field.bytes().await.map_err(|err| {
            tracing::error!(error = ?err,"无法读取字节");
            Error::BadRequest("无法读取字节".into())
        })?;

        // 创建一个文件夹以存储上传的文件
        let now = chrono::offset::Local::now()
            .format("%Y%m%d_%H%M%S")
            .to_string();
        let uuid = uuid::Uuid::new_v4().to_string();
        let folder = format!("{now}_{uuid}");
        let upload_folder = PathBuf::from(UPLOAD_DIR).join(&folder);
        fs::create_dir_all(&upload_folder).await?;

        // 将文件写入新创建的文件夹
        let path = upload_folder.join(file_name);
        let mut f = fs::OpenOptions::new()
            .create_new(true)
            .write(true)
            .open(&path)
            .await?;
        f.write_all(&content).await?;
        f.flush().await?;

        // 在数据库中记录文件上传
        let file = files::ActiveModel {
            notes_id: ActiveValue::Set(notes_id),
            file_path: ActiveValue::Set(
                path.strip_prefix(UPLOAD_DIR)
                    .unwrap()
                    .to_str()
                    .unwrap()
                    .to_string(),
            ),
            ..Default::default()
        }
        .insert(&ctx.db)
        .await?;

        files.push(file);
    }

    format::json(files)
}

pub async fn list(
    _auth: auth::JWT,
    Path(notes_id): Path<i32>,
    State(ctx): State<AppContext>,
) -> Result<Response> {
    // 获取为特定笔记上传的所有文件
    let files = files::Entity::find()

查看文件​

查看特定的 files。

loco_starter/src/controllers/files.rs
#[debug_handler]
pub async fn view(
    _auth: auth::JWT,
    Path(files_id): Path<i32>,
    State(ctx): State<AppContext>,
) -> Result<Response> {
    // 从数据库中获取文件信息
    let file = files::Entity::find_by_id(files_id)
        .one(&ctx.db)
        .await?
        .expect("未找到文件");

    // 流式传输文件
    let file = fs::File::open(format!("{UPLOAD_DIR}/{}", file.file_path)).await?;
    let stream = ReaderStream::new(file);
    let body = Body::from_stream(stream);

    Ok(format::render().response().body(body)?)
}

SQL Server 支持​

SeaORM 的 SQL Server 现已提供封闭测试。如果你感兴趣，请在 这里 注册。

从 sea-orm 迁移到 sea-orm-x 非常简单，只需两个简单步骤。首先，将现有的 sea-orm 依赖项更新为 sea-orm-x 并启用 sqlz-mssql 功能。请注意，您可能需要修补 SeaORM 依赖项以解决上游依赖项的问题。

Cargo.toml
sea-orm = { path = "<SEA_ORM_X_ROOT>/sea-orm-x", features = ["runtime-async-std-rustls", "sqlz-mssql"] }
sea-orm-migration = { path = "<SEA_ORM_X_ROOT>/sea-orm-x/sea-orm-migration" }

# 修补 SeaORM 依赖项以解决上游依赖项
[patch.crates-io]
sea-orm = { path = "<SEA_ORM_X_ROOT>/sea-orm-x" }
sea-orm-migration = { path = "<SEA_ORM_X_ROOT>/sea-orm-x/sea-orm-migration" }

其次，更新连接字符串以连接到 MSSQL 数据库。

# 如果模式是 `dbo`，只需编写：
mssql://username:password@host/database

# 或者，通过提供额外的 `currentSchema` 查询参数来指定模式名称。
mssql://username:password@host/database?currentSchema=my_schema

# 通过提供额外的 trustCertificate 查询参数来信任同行证书。
mssql://username:password@host/database?trustCertificate=true
