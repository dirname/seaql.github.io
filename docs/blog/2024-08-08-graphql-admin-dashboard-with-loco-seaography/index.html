基于 GraphQL 的 Loco 和 Seaography 管理仪表盘

August 8, 2024 · 9 min read
Billy Chan
SeaQL 团队

在本教程中，我们将使用 Seaography 和 Loco 开发一个基于 GraphQL 的管理仪表盘。

如果你还没有阅读我们的系列教程的第一部分和第二部分，请查看：使用 Loco 和 SeaORM 入门、使用 Seaography 为 Loco 添加 GraphQL 支持。

完整的源代码可以在 这里 找到。

什么是 Seaography​

Seaography 是一个用于构建 GraphQL 解析器的 GraphQL 框架，它使用 SeaORM 实体。它附带一个 CLI 工具，可以从现有的 MySQL、Postgres 和 SQLite 数据库生成可编译的 Rust GraphQL 服务器。

设置 React-Admin 前端模板​

我们使用 React-Admin 作为前端框架。它提供了一个方便的模板来开始：

$ npm init react-admin frontend

运行模板，然后访问 http://localhost:5173/，你应该可以看到欢迎页面：

$ cd frontend

$ npm install

$ npm run dev

现在，我们希望使用模拟数据展示 React-Admin 数据表模板。首先，我们需要添加 ra-data-json-server 依赖，它提供了一个现成的模拟数据加载器：

$ npm install ra-data-json-server

为准备模拟数据加载器，我们创建一个新文件：

frontend/src/dataProvider.ts
import jsonServerProvider from 'ra-data-json-server';

export const dataProvider = jsonServerProvider('https://jsonplaceholder.typicode.com');

接下来，我们更改 UI 文件：

frontend/src/index.tsx
+ import { Admin, Resource, ListGuesser, ShowGuesser } from 'react-admin';

);

现在运行模板，你应该能看到用户列表页面：

$ npm run dev

点击每一行以查看其详细信息页面。

添加 NPM 依赖​

接下来，我们开始将 Loco 和 Seaography 后端与 React-Admin 前端集成。我们使用 axios 发送 POST 请求到我们的 GraphQL 后端：

$ npm install axios

GraphQL 数据提供者​

然后，我们可以通过替换 dataProvider.ts 的内容来开始实现 GraphQL 数据提供者：

frontend/src/dataProvider.ts
- import jsonServerProvider from 'ra-data-json-server';
- 
- export const dataProvider = jsonServerProvider('https://jsonplaceholder.typicode.com');

将其集成到我们的 GraphQL 端点 http://localhost:3000/api/graphql。我们实现了两个处理器，一个用于获取帖子列表数据，另一个用于获取单个帖子的详细数据：

frontend/src/dataProvider.ts
import { DataProvider } from "react-admin";
import axios from 'axios';

const apiUrl = 'http://localhost:3000/api/graphql';

export const dataProvider: DataProvider = {
    // 获取帖子的列表数据
    getList: (resource, params) => {
        // 分页状态
        const { page, perPage } = params.pagination;
        // 排序状态
        const { field, order } = params.sort;

        // 向 GraphQL 端点发送 POST 请求
        return axios.post(apiUrl, {
            query: `
            query {
              notes (
                orderBy: { ${field}: ${order} },
                pagination: { page: { limit: ${perPage}, page: ${page - 1} }}
              ) {
                nodes {
                  id
                  title
                  createdAt
                  updatedAt
                }
                paginationInfo {
                  pages
                  current
                  offset
                  total
                }
              }
            }
            `
        })
            .then((response) => {
                // 解封响应
                const { nodes, paginationInfo } = response.data.data.notes;
                // 返回数据数组和总页数
                return {
                    data: nodes,
                    total: paginationInfo.total,
                };
            });
    },

    // 获取单个帖子的详细数据
    getOne: (resource, params) => {
        // 向 GraphQL 端点发送 POST 请求
        return axios.post(apiUrl, {
            query: `
            query {
              notes(filters: {id: {eq: ${params.id}}}) {
                nodes {
                  id
                  title
                  content
                  createdAt
                  updatedAt
                }
              }
            }
            `
        })
            .then((response) => {
                // 解封响应
                const { nodes } = response.data.data.notes;
                // 返回唯一数据
                return {
                    data: nodes[0],
                };
            });
    },

    getMany: (resource, params) => { },

    getManyReference: (resource, params) => { },

    update: (resource, params) => { },

    updateMany: (resource, params) => { },

    create: (resource, params) => { },

    delete: (resource, params) => { },

    deleteMany: (resource, params) => { },
};

自定义 React-Admin 前端​

用我们自己的自定义 UI 替换 React-Admin 模板前端，以列出来自数据库的所有笔记。

frontend/src/index.tsx
- ReactDOM.createRoot(document.getElementById('root')!).render(
-    <Admin dataProvider={dataProvider}>
-        <Resource name="users" list={ListGuesser} show={ShowGuesser} />
-    </Admin>
- );

使用特定列实现列表和详情页面：

frontend/src/index.tsx
import ReactDOM from 'react-dom/client';
import { Admin, Resource, List, Datagrid, TextField, Show, SimpleShowLayout } from 'react-admin';
import { dataProvider } from './dataProvider';

const PostList = () => (
    <List>
        <Datagrid bulkActionButtons={false}>
            <TextField source="id" />
            <TextField source="title" />
            <TextField source="content" />
            <TextField source="createdAt" />
            <TextField source="updatedAt" />
        </Datagrid>
    </List>
);

const PostShow = () => (
    <Show>
        <SimpleShowLayout>
            <TextField source="id" />
            <TextField source="title" />
            <TextField source="content" />
            <TextField source="createdAt" />
            <TextField source="updatedAt" />
        </SimpleShowLayout>
    </Show>
);

ReactDOM.createRoot(document.getElementById('root')!).render(
    <Admin dataProvider={dataProvider}>
        <Resource name="posts" list={PostList} show={PostShow} />
    </Admin>
);

无认证的 GraphQL 端点​

为了方便起见，禁用了 GraphQL POST 处理端点上的用户身份验证：

backend/src/controllers/graphql.rs
async fn graphql_handler(
-   _auth: auth::JWT,
    State(ctx): State<AppContext>,
    req: Request<Body>,
) -> Result<Response> {
    const DEPTH: usize = 1_000;
    const COMPLEXITY: usize = 1_000;
    // 构造 GraphQL 查询根
    let schema = query_root::schema(ctx.db.clone(), DEPTH, COMPLEXITY).unwrap();
    // GraphQL 处理器
    let mut graphql_handler = async_graphql_axum::GraphQL::new(schema);
    // 执行 GraphQL 请求并获取结果
    let res = graphql_handler.call(req).await.unwrap();

    Ok(res)
}

结论​

借助 Seaography，为 Loco 应用添加 GraphQL 支持是非常简单的。它是一个符合人体工程学的库，完美地与任何前端框架集成。本教程仅涵盖 LOco 和 Seaography 的基本集成，包括仅通过 GraphQL 端点查询数据。未演示 GraphQL 变更，我们留给您自行编码！

SQL Server 支持​

适用于 SeaORM 的 SQL Server 现已作为封闭测试版发布。如果你感兴趣，请在 这里 注册。

从 sea-orm 迁移到 sea-orm-x 有两个简单步骤。首先，将现有的 sea-orm 依赖更新为 sea-orm-x 并启用 sqlz-mssql 功能。请注意，您可能需要补丁 SeaORM 依赖以适应上游依赖。

Cargo.toml
sea-orm = { path = "<SEA_ORM_X_ROOT>/sea-orm-x", features = ["runtime-async-std-rustls", "sqlz-mssql"] }
sea-orm-migration = { path = "<SEA_ORM_X_ROOT>/sea-orm-x/sea-orm-migration" }

# 修补 SeaORM 依赖以适应上游依赖
[patch.crates-io]
sea-orm = { path = "<SEA_ORM_X_ROOT>/sea-orm-x" }
sea-orm-migration = { path = "<SEA_ORM_X_ROOT>/sea-orm-x/sea-orm-migration" }

第二，更新连接字符串以连接到 MSSQL 数据库。

# 如果模式为 `dbo`，只需写：
mssql://username:password@host/database

# 或通过提供额外的 `currentSchema` 查询参数指定模式名称。
mssql://username:password@host/database?currentSchema=my_schema

# 通过提供额外的 trustCertificate 查询参数，您可以信任对等证书。
mssql://username:password@host/database?trustCertificate=true
