SeaORM 0.6.0 新特性

February 7, 2022 · 6 min read
SeaQL Team
Chris Tsang

🎉 我们很高兴地宣布今天发布了 SeaORM 0.6.0！以下是一些特性亮点 🌟：

数据迁移​

[#335] 使用 SeaQuery 或原始 SQL 编写的迁移来进行数据库架构的版本控制。查看 迁移文档 了解更多信息。

  1. 通过执行 sea-orm-cli migrate init 设置迁移目录。

    migration
    ├── Cargo.toml
    ├── README.md
    └── src
        ├── lib.rs
        ├── m20220101_000001_create_table.rs
        └── main.rs

  2. 在 SeaQuery 中定义迁移。

    use sea_schema::migration::prelude::*;

    pub struct Migration;

    impl MigrationName for Migration {
        fn name(&self) -> &str {
            "m20220101_000001_create_table"
        }
    }

    #[async_trait::async_trait]
    impl MigrationTrait for Migration {
        async fn up(&self, manager: &SchemaManager) -> Result<(), DbErr> {
            manager
                .create_table( ... )
                .await
        }

        async fn down(&self, manager: &SchemaManager) -> Result<(), DbErr> {
            manager
                .drop_table( ... )
                .await
        }
    }

  3. 通过执行 sea-orm-cli migrate 应用迁移。

    $ sea-orm-cli migrate
    应用所有待处理的迁移
    应用迁移 'm20220101_000001_create_table'
    迁移 'm20220101_000001_create_table' 已成功应用

模型中支持 DateTimeUtc 和 DateTimeLocal​

[#489] 在模型中用 DateTimeLocal (chrono::DateTime<Local>) 或 DateTimeUtc (chrono::DateTime<Utc>) 类型属性表示数据库的时间戳列。

#[derive(Clone, Debug, PartialEq, DeriveEntityModel)]
#[sea_orm(table_name = "satellite")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub satellite_name: String,
    pub launch_date: DateTimeUtc,
    pub deployment_date: DateTimeLocal,
}

模拟连接结果​

[#455] 构建相关模型的模拟结果，使用模型的元组。

let db = MockDatabase::new(DbBackend::Postgres)
    // 模拟蛋糕与其相关水果的结果
    .append_query_results(vec![vec![(
        cake::Model {
            id: 1,
            name: "苹果蛋糕".to_owned(),
        },
        fruit::Model {
            id: 2,
            name: "苹果".to_owned(),
            cake_id: Some(1),
        },
    )]])
    .into_connection();

assert_eq!(
    cake::Entity::find()
        .find_also_related(fruit::Entity)
        .all(&db)
        .await?,
    vec![(
        cake::Model {
            id: 1,
            name: "苹果蛋糕".to_owned(),
        },
        Some(fruit::Model {
            id: 2,
            name: "苹果".to_owned(),
            cake_id: Some(1),
        })
    )]
);

支持最大连接生命周期选项​

[#493] 您可以使用 max_lifetime 方法设置单个连接的最大生命周期。

let mut opt = ConnectOptions::new("protocol://username:password@host/database".to_owned());
opt.max_lifetime(Duration::from_secs(8))
    .max_connections(100)
    .min_connections(5)
    .connect_timeout(Duration::from_secs(8))
    .idle_timeout(Duration::from_secs(8))
    .sqlx_logging(true);

let db = Database::connect(opt).await?;
