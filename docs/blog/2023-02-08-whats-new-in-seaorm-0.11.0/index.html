SeaORM 0.11.0 的新功能

February 8, 2023 · 12 min read
SeaQL 团队
Chris Tsang

🎉 我们很高兴发布 SeaORM 0.11.0!

数据加载器​

[[#1443](https://github.com/SeaQL/sea-orm/pull/1443), [#1238](https://github.com/SeaQL/sea-orm/pull/1238)][LoaderTrait](https://docs.rs/sea-orm/*/sea_orm/query/trait.LoaderTrait.html) 提供了一个 API，用于批量加载相关实体。

考虑这种一对多关系：

let cake_with_fruits: Vec<(cake::Model, Vec<fruit::Model>)> = Cake::find()
    .find_with_related(Fruit)
    .all(db)
    .await?;

生成的 SQL 为：

SELECT
    "cake"."id" AS "A_id",
    "cake"."name" AS "A_name",
    "fruit"."id" AS "B_id",
    "fruit"."name" AS "B_name",
    "fruit"."cake_id" AS "B_cake_id"
FROM "cake"
LEFT JOIN "fruit" ON "cake"."id" = "fruit"."cake_id"
ORDER BY "cake"."id" ASC

1 端（蛋糕）的数据将会重复。如果 N 的值很大，这将导致更多的数据通过网络传输。使用 Loader 将确保每个模型仅传输一次。

以下代码加载与上面相同的数据，但使用了两个查询：

let cakes: Vec<cake::Model> = Cake::find().all(db).await?;
let fruits: Vec<Vec<fruit::Model>> = cakes.load_many(Fruit, db).await?;

for (cake, fruits) in cakes.into_iter().zip(fruits.into_iter()) { .. }
SELECT "cake"."id", "cake"."name" FROM "cake"
SELECT "fruit"."id", "fruit"."name", "fruit"."cake_id" FROM "fruit" WHERE "fruit"."cake_id" IN (..)

您甚至可以在相关实体上应用过滤器：

let fruits_in_stock: Vec<Vec<fruit::Model>> = cakes.load_many(
    fruit::Entity::find().filter(fruit::Column::Stock.gt(0i32)),
    db
).await?;
SELECT "fruit"."id", "fruit"."name", "fruit"."cake_id" FROM "fruit"
WHERE "fruit"."stock" > 0 AND "fruit"."cake_id" IN (..)

要了解更多信息，请阅读 关系文档。

事务隔离级别和访问模式​

[[#1230](https://github.com/SeaQL/sea-orm/pull/1230)][`transaction_with_config`](https://docs.rs/sea-orm/*/sea_orm/trait.TransactionTrait.html#tymethod.transaction_with_config) 和 begin_with_config 允许您指定 IsolationLevel 和 AccessMode。

目前，它们仅在 MySQL 和 Postgres 中实现。为了对齐它们的语义差异，MySQL 会在开始事务之前执行 SET TRANSACTION 命令，而 Postgres 则在开始事务之后执行 SET TRANSACTION 命令。

db.transaction_with_config::<_, _, DbErr>(
    |txn| { ... },
    Some(IsolationLevel::ReadCommitted),
    Some(AccessMode::ReadOnly),
)
.await?;

let transaction = db
    .begin_with_config(IsolationLevel::ReadCommitted, AccessMode::ReadOnly)
    .await?;

要了解更多信息，请阅读 事务文档。

在选择和保存时转换列类型​

[#1304] 如果您需要选择一个列为一种类型，但将其保存为另一种类型，您可以通过指定 select_as 和 save_as 属性来完成转换。一个典型的用例是选择 citext（区分大小写的文本）类型的列作为 Rust 中的 String，并将其保存为 citext。您应如下定义模型字段：

#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "ci_table")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    #[sea_orm(select_as = "text", save_as = "citext")]
    pub case_insensitive_text: String
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {}

impl ActiveModelBehavior for ActiveModel {}

对 ActiveModelBehavior 的更改​

[#1328, #1145] ActiveModelBehavior 的方法现在有一个 Connection 作为额外参数。这使您能够执行数据库操作，例如，记录对现有模型所做的更改或在插入之前验证数据。

#[async_trait]
impl ActiveModelBehavior for ActiveModel {
    /// 创建一个具有默认值的新 ActiveModel，也用于 `Default::default()`。
    fn new() -> Self {
        Self {
            uuid: Set(Uuid::new_v4()),
            ..ActiveModelTrait::default()
        }
    }

    /// 将在插入/更新之前触发
    async fn before_save<C>(self, db: &C, insert: bool) -> Result<Self, DbErr>
    where
        C: ConnectionTrait,
    {
        // 记录变更
        edit_log::ActiveModel {
            action: Set("before_save".into()),
            values: Set(serde_json::json!(model)),
            ..Default::default()
        }
        .insert(db)
        .await?;

        Ok(self)
    }
}

要了解更多信息，请阅读 实体文档。

执行未准备的 SQL 语句​

[#1327] 您可以使用 ConnectionTrait::execute_unprepared 执行未准备的 SQL 语句。

// 如果 SQL 语句没有值绑定，则使用 `execute_unprepared`
db.execute_unprepared(
    "CREATE TABLE `cake` (
        `id` int NOT NULL AUTO_INCREMENT PRIMARY KEY,
        `name` varchar(255) NOT NULL
    )"
)
.await?;

// 如果 SQL 包含值绑定，则构造 `Statement`
let stmt = Statement::from_sql_and_values(
    manager.get_database_backend(),
    r#"INSERT INTO `cake` (`name`) VALUES (?)"#,
    ["Cheese Cake".into()]
);
db.execute(stmt).await?;

选择到元组​

[#1311] 您可以使用 into_tuple 方法选择元组（或单个值）。

let res: Vec<(String, i64)> = cake::Entity::find()
    .select_only()
    .column(cake::Column::Name)
    .column(cake::Column::Id.count())
    .group_by(cake::Column::Name)
    .into_tuple()
    .all(&db)
    .await?;

原子迁移​

[#1379] 迁移将在 Postgres 中原子执行，这意味着迁移脚本将在事务中执行。如果迁移失败，对数据库所做的更改将回滚。然而，MySQL 和 SQLite 不支持原子迁移。

您可以在每个迁移内启动一个事务，以执行诸如为新创建的表 填充示例数据 的操作。

类型支持​

  • [#1325] 支持 uuid::fmt 模块中可用的各种 UUID 格式
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "uuid_fmt")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub uuid: Uuid,
    pub uuid_braced: uuid::fmt::Braced,
    pub uuid_hyphenated: uuid::fmt::Hyphenated,
    pub uuid_simple: uuid::fmt::Simple,
    pub uuid_urn: uuid::fmt::Urn,
}
  • [#1210] 对 Postgres 的枚举向量支持
#[derive(Debug, Clone, PartialEq, Eq, EnumIter, DeriveActiveEnum)]
#[sea_orm(rs_type = "String", db_type = "Enum", enum_name = "tea")]
pub enum Tea {
    #[sea_orm(string_value = "EverydayTea")]
    EverydayTea,
    #[sea_orm(string_value = "BreakfastTea")]
    BreakfastTea,
}

#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "enum_vec")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub teas: Vec<Tea>,
    pub teas_opt: Option<Vec<Tea>>,
}
  • [#1414] 支持将 ActiveEnum 字段用作主键
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "enum_primary_key")]
pub struct Model {
    #[sea_orm(primary_key, auto_increment = false)]
    pub id: Tea,
    pub category: Option<Category>,
    pub color: Option<Color>,
}

可选择的不稳定内部 API​

通过启用 sea-orm-internal 功能，您可以选择不稳定的内部 API，包括：

  • 使用 get_*_connection_pool 方法访问 SQLx 的内部连接池
  • 重新导出 SQLx 错误类型：SqlxError、SqlxMySqlError、SqlxPostgresError 和 SqlxSqliteError

破坏性更改​

  • [#1366] sea-query 已升级到 0.28.x，这带来了一些改进和破坏性更改。有关详细信息，请参阅发布说明

  • [#1420] sea-orm-cli：generate entity 命令默认启用 --universal-time 标志

  • [#1425] 向 DbErr 添加 RecordNotInserted 和 RecordNotUpdated

  • [#1327] 添加了 ConnectionTrait::execute_unprepared 方法

  • [#1311] TryGetable 的要求方法已更改：

// 然后
fn try_get(res: &QueryResult, pre: &str, col: &str) -> Result<Self, TryGetError>;
// 现在; ColIdx 可以是 `&str` 或 `usize`
fn try_get_by<I: ColIdx>(res: &QueryResult, index: I) -> Result<Self, TryGetError>;

因此，如果您自己实现了它：

impl TryGetable for XXX {
-   fn try_get(res: &QueryResult, pre: &str, col: &str) -> Result<Self, TryGetError> {
+   fn try_get_by<I: sea_orm::ColIdx>(res: &QueryResult, idx: I) -> Result<Self, TryGetError> {
-       let value: YYY = res.try_get(pre, col).map_err(TryGetError::DbErr)?;
+       let value: YYY = res.try_get_by(idx).map_err(TryGetError::DbErr)?;
        ..
    }
}
  • [#1328] ActiveModelBehavior 特性变为异步特性。 如果您重写了默认 ActiveModelBehavior 的实现：
#[async_trait::async_trait]
impl ActiveModelBehavior for ActiveModel {
    async fn before_save<C>(self, db: &C, insert: bool) -> Result<Self, DbErr>
    where
        C: ConnectionTrait,
    {
        // ...
    }

    // ...
}
  • [#1425] DbErr::RecordNotFound("没有影响数据库行") 被移至专用错误变体 DbErr::RecordNotUpdated
let res = Update::one(cake::ActiveModel {
        name: Set("Cheese Cake".to_owned()),
        ..model.into_active_model()
    })
    .exec(&db)
    .await;

// 然后
assert_eq!(
    res,
    Err(DbErr::RecordNotFound(
        "没有影响数据库行".to_owned()
    ))
);

// 现在
assert_eq!(res, Err(DbErr::RecordNotUpdated));
  • [#1395] sea_orm::ColumnType 被替换为 sea_query::ColumnType
    • 方法 ColumnType::def 被移动到 ColumnTypeTrait
    • ColumnType::Binary 成为一个元组变体，接受额外选项 sea_query::BlobSize
    • ColumnType::Custom 使用 sea_query::DynIden 而不是 String，并因此添加了一个新方法 custom（注意小写）
// 精简实体
#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "fruit")]
pub struct Model {
-   #[sea_orm(column_type = r#"Custom("citext".to_owned())"#)]
+   #[sea_orm(column_type = r#"custom("citext")"#)]
    pub column: String,
}
// 扩展实体
impl ColumnTrait for Column {
    type EntityName = Entity;

    fn def(&self) -> ColumnDef {
        match self {
-           Self::Column => ColumnType::Custom("citext".to_owned()).def(),
+           Self::Column => ColumnType::custom("citext").def(),
        }
    }
}

SeaORM 增强​

  • [#1256] 重构 schema 模块以公开数据库修改功能
  • [#1346] 使用 #[sea_orm(column_type = "JsonBinary")] 宏属性生成紧凑实体
  • MockDatabase::append_exec_results()、MockDatabase::append_query_results()、MockDatabase::append_exec_errors() 和 MockDatabase::append_query_errors() [#1367] 接受任何实现了 IntoIterator 特性的类型
  • [#1362] find_by_id 和 delete_by_id 接受任何 Into 主键值
  • [#1410] QuerySelect::offset 和 QuerySelect::limit 接受 Into<Option<u64>>，其中 None 将重置它们
  • [#1236] 添加了 DatabaseConnection::close
  • [#1381] 为 ColumnDef 添加了 is_null getter
  • [#1177] 添加了 ActiveValue::reset 用于将 Unchanged 转换为 Set
  • [#1415] 添加了 QueryTrait::apply_if 可选择地应用过滤器
  • 添加了 sea-orm-internal 功能标志以公开某些 SQLx 类型
    • [#1297] 添加了 DatabaseConnection::get_*_connection_pool() 用于访问内部 SQLx 连接池
    • [#1434] 重新导出 SQLx 错误
