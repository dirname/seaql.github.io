使用 SQLite

如果您想测试不需要特定于数据库的功能的应用程序逻辑，SQLite 将是一个不错的选择。

请查看简单示例 这里。

集成测试​

建议在 集成测试 中执行更复杂的测试用例。以下代码片段展示了连接到数据库、设置模式以及执行测试的步骤。

async fn main() -> Result<(), DbErr> {
    // 连接到 SQLite
    let db = Database::connect("sqlite::memory:").await?;

    // 设置数据库模式
    setup_schema(&db).await?;

    // 执行测试
    testcase(&db).await?;

    Ok(())
}

设置数据库模式​

要在 SQLite 数据库中创建供测试使用的表，而不是手动编写 TableCreateStatement，您可以使用 Schema::create_table_from_entity 从 Entity 派生。

async fn setup_schema(db: &DbConn) {

    // 设置模式助手
    let schema = Schema::new(DbBackend::Sqlite);

    // 从 Entity 派生
    let stmt: TableCreateStatement = schema.create_table_from_entity(MyEntity);

    // 或手动设置
    assert_eq!(
        stmt.build(SqliteQueryBuilder),
        Table::create()
            .table(MyEntity)
            .col(
                ColumnDef::new(MyEntity::Column::Id)
                    .integer()
                    .not_null()
            )
            //...
            .build(SqliteQueryBuilder)
    );

    // 执行创建表语句
    let result = db
        .execute(db.get_database_backend().build(&stmt))
        .await;
}

执行测试​

执行测试用例并对结果进行断言。

async fn testcase(db: &DbConn) -> Result<(), DbErr> {

    let baker_bob = baker::ActiveModel {
        name: Set("Baker Bob".to_owned()),
        contact_details: Set(serde_json::json!({
            "mobile": "+61424000000",
            "home": "0395555555",
            "address": "12 Test St, Testville, Vic, Australia"
        })),
        bakery_id: Set(2),
        ..Default::default()
    };

    let baker_insert_res = Baker::insert(baker_bob)
        .exec(db)
        .await
        .expect("无法插入面包师");

    assert_eq!(baker_insert_res.last_insert_id, 1);

    Ok(())
}
