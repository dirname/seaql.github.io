插入

在深入了解 SeaORM 的插入功能之前，我们需要介绍 ActiveValue 和 ActiveModel。

ActiveValue​

用于捕获对 ActiveModel 属性所做更改的包装结构体。

use sea_orm::ActiveValue::{Set, NotSet, Unchanged};

// 设置值
let _: ActiveValue<i32> = Set(10);

// 未设置值
let _: ActiveValue<i32> = NotSet;

// 一个 `未更改` 的值
let v: ActiveValue<i32> = Unchanged(10);

// 将 `未更改` 的活动值转换为 `设置`
assert!(v.reset(), Set(10));

模型与 ActiveModel​

ActiveModel 拥有 Model 所有属性，并用 ActiveValue 包装。

你可以使用 ActiveModel 在设置了一部分列的情况下插入一行。

let cheese: Option<cake::Model> = Cake::find_by_id(1).one(db).await?;

// 获取模型
let model: cake::Model = cheese.unwrap();
assert_eq!(model.name, "Cheese Cake".to_owned());

// 转换为 ActiveModel
let active_model: cake::ActiveModel = model.into();
assert_eq!(active_model.name, ActiveValue::unchanged("Cheese Cake".to_owned()));

从 JSON 值设置 ActiveModel​

如果你想将用户输入保存到数据库，可以轻松地将 JSON 值转换为 ActiveModel。请注意，你可能希望跳过反序列化 JSON 的主键属性，你可以按如下所示进行配置。

#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel, Serialize, Deserialize)]
#[sea_orm(table_name = "fruit")]
pub struct Model {
    #[sea_orm(primary_key)]
    #[serde(skip_deserializing)] // 跳过反序列化
    pub id: i32,
    pub name: String,
    pub cake_id: Option<i32>,
}

使用 set_from_json 方法在 ActiveModel 中设置属性。

// 一个设置了主键的 ActiveModel
let mut fruit = fruit::ActiveModel {
    id: ActiveValue::Set(1),
    name: ActiveValue::NotSet,
    cake_id: ActiveValue::NotSet,
};

// 请注意，此方法不会改变 ActiveModel 中的主键值
fruit.set_from_json(json!({
    "id": 8,
    "name": "Apple",
    "cake_id": 1,
}))?;

assert_eq!(
    fruit,
    fruit::ActiveModel {
        id: ActiveValue::Set(1),
        name: ActiveValue::Set("Apple".to_owned()),
        cake_id: ActiveValue::Set(Some(1)),
    }
);

使用 from_json 方法根据 JSON 值创建一个新的 ActiveModel。

let fruit = fruit::ActiveModel::from_json(json!({
    "name": "Apple",
}))?;

assert_eq!(
    fruit,
    fruit::ActiveModel {
        id: ActiveValue::NotSet,
        name: ActiveValue::Set("Apple".to_owned()),
        cake_id: ActiveValue::NotSet,
    }
);

检查 ActiveModel 是否已更改​

你可以使用 is_changed 方法检查 ActiveModel 中的任何字段是否为 Set。

let mut fruit: fruit::ActiveModel = Default::default();
assert!(!fruit.is_changed());

fruit.set(fruit::Column::Name, "apple".into());
assert!(fruit.is_changed());

将 ActiveModel 转换回 Model​

使用 try_into_model 方法可以将 ActiveModel 转换回 Model。

assert_eq!(
    ActiveModel {
        id: Set(2),
        name: Set("Apple".to_owned()),
        cake_id: Set(Some(1)),
    }
    .try_into_model()
    .unwrap(),
    Model {
        id: 2,
        name: "Apple".to_owned(),
        cake_id: Some(1),
    }
);

assert_eq!(
    ActiveModel {
        id: Set(1),
        name: NotSet,
        cake_id: Set(None),
    }
    .try_into_model(),
    Err(DbErr::AttrNotSet(String::from("name")))
);

插入一条记录​

插入一个活动模型并返回一个新的 Model。其值从数据库中检索，因此任何自动生成的字段将被填充。

let pear = fruit::ActiveModel {
    name: Set("Pear".to_owned()),
    ..Default::default() // 所有其他属性为 `NotSet`
};

let pear: fruit::Model = pear.insert(db).await?;

插入一个活动模型并返回最后插入的 ID。其类型与模型的主键类型匹配，因此如果模型具有复合主键，则可能是一个元组。

let pear = fruit::ActiveModel {
    name: Set("Pear".to_owned()),
    ..Default::default() // 所有其他属性为 `NotSet`
};

let res: InsertResult = fruit::Entity::insert(pear).exec(db).await?;
assert_eq!(res.last_insert_id, 28)

插入多条记录​

插入多个活动模型并返回最后插入的 ID。

let apple = fruit::ActiveModel {
    name: Set("Apple".to_owned()),
    ..Default::default()
};

let orange = fruit::ActiveModel {
    name: Set("Orange".to_owned()),
    ..Default::default()
};

let res: InsertResult = Fruit::insert_many([apple, orange]).exec(db).await?;
assert_eq!(res.last_insert_id, 30)

向 insert_many 方法提供一个空集将导致错误。然而，你可以通过 on_empty_do_nothing 更改行为，它会将 InsertResult 包装在 TryInsertResult 中。

let res = Bakery::insert_many(std::iter::empty())
    .on_empty_do_nothing()
    .exec(db)
    .await;

assert!(matches!(res, Ok(TryInsertResult::Empty)));

处理冲突​

插入活动模型时执行冲突处理行为。

let orange = cake::ActiveModel {
    id: ActiveValue::set(2),
    name: ActiveValue::set("Orange".to_owned()),
};

assert_eq!(
    cake::Entity::insert(orange.clone())
        .on_conflict(
            // 冲突时不做任何处理
            sea_query::OnConflict::column(cake::Column::Name)
                .do_nothing()
                .to_owned()
        )
        .build(DbBackend::Postgres)
        .to_string(),
    r#"INSERT INTO "cake" ("id", "name") VALUES (2, 'Orange') ON CONFLICT ("name") DO NOTHING"#,
);

assert_eq!(
    cake::Entity::insert(orange)
        .on_conflict(
            // 冲突时进行更新
            sea_query::OnConflict::column(cake::Column::Name)
                .update_column(cake::Column::Name)
                .to_owned()
        )
        .build(DbBackend::Postgres)
        .to_string(),
    r#"INSERT INTO "cake" ("id", "name") VALUES (2, 'Orange') ON CONFLICT ("name") DO UPDATE SET "name" = "excluded"."name""#,
);

如果在未插入或更新任何行时执行 upsert 语句，将导致 DbErr::RecordNotInserted 错误。

// 当 `id` 列具有冲突值时，不执行任何操作
let on_conflict = OnConflict::column(Column::Id).do_nothing().to_owned();

// 将 `1`, `2`, `3` 插入到表中
let res = Entity::insert_many([
    ActiveModel { id: Set(1) },
    ActiveModel { id: Set(2) },
    ActiveModel { id: Set(3) },
])
.on_conflict(on_conflict.clone())
.exec(db)
.await;

assert_eq!(res?.last_insert_id, 3);

// 将 `4` 插入到表中，同时添加之前的 3 行
let res = Entity::insert_many([
    ActiveModel { id: Set(1) },
    ActiveModel { id: Set(2) },
    ActiveModel { id: Set(3) },
    ActiveModel { id: Set(4) },
])
.on_conflict(on_conflict.clone())
.exec(db)
.await;

assert_eq!(res?.last_insert_id, 4);

// 重复最后的插入。由于所有 4 行已经存在，这实际上什么也没做。
// 将抛出 `DbErr::RecordNotInserted` 错误。
let res = Entity::insert_many([
    ActiveModel { id: Set(1) },
    ActiveModel { id: Set(2) },
    ActiveModel { id: Set(3) },
    ActiveModel { id: Set(4) },
])
.on_conflict(on_conflict)
.exec(db)
.await;

assert_eq!(res.err(), Some(DbErr::RecordNotInserted));

如果你希望 RecordNotInserted 成为一个 Ok 而不是错误，请调用 .do_nothing()：

let res = Entity::insert_many([..])
    .on_conflict(on_conflict)
    .do_nothing()
    .exec(db)
    .await;

assert!(matches!(res, Ok(TryInsertResult::Conflicted)));
