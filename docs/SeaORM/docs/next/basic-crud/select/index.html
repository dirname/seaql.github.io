Select

一旦定义了实体，您就可以开始从数据库中检索数据。数据库中的每一行数据对应一个 Model。

默认情况下，SeaORM 将选择在 Column 枚举中定义的所有列。

按主键查找​

通过主键查找模型，可以是单个主键或复合主键。我们通过调用 find_by_id 在 Entity 上开始，此方法会自动帮助您构建选择查询和条件。然后，使用 one 方法从数据库中获取一个模型。

use super::cake::Entity as Cake;
use super::cake_filling::Entity as CakeFilling;

// 按主键查找
let cheese: Option<cake::Model> = Cake::find_by_id(1).one(db).await?;

// 按复合主键查找
let vanilla: Option<cake_filling::Model> = CakeFilling::find_by_id((6, 8)).one(db).await?;

按条件和顺序查找​

除了通过主键检索模型外，您还可以检索一个或多个符合特定条件并按照特定顺序排列的模型。 find 方法让您可以访问 SeaORM 中的查询构建器。它支持构建所有常见的选择表达式，例如 where 和 order by。可以分别使用 filter 和 order_by_* 方法构造。

阅读更多关于 条件表达式 的内容。

let chocolate: Vec<cake::Model> = Cake::find()
    .filter(cake::Column::Name.contains("chocolate")) // 过滤包含“chocolate”的蛋糕
    .order_by_asc(cake::Column::Name) // 按名称升序排列
    .all(db)
    .await?;

查找相关模型​

阅读更多关于 关系 的章节。

延迟加载​

使用 find_related 方法。

相关模型在您请求时按需加载，如果您希望根据某些应用逻辑加载相关模型，这种方式更可取。请注意，与积极加载相比，延迟加载会增加数据库的往返次数。

// 首先查找一个蛋糕模型
let cheese: Option<cake::Model> = Cake::find_by_id(1).one(db).await?;
let cheese: cake::Model = cheese.unwrap(); // 解包

// 然后，查找此蛋糕的所有相关水果
let fruits: Vec<fruit::Model> = cheese.find_related(Fruit).all(db).await?;

积极加载​

所有相关模型一次性加载。这相比于延迟加载提供了最少的数据库往返次数。

一对一​

使用 find_also_related 方法。

let fruits_and_cakes: Vec<(fruit::Model, Option<cake::Model>)> = Fruit::find().find_also_related(Cake).all(db).await?;

一对多 / 多对多​

使用 find_with_related 方法，相关模型将按父模型分组。该方法处理 1-N 和 M-N 情况，当涉及交叉表时将执行 2 次连接。

let cake_with_fruits: Vec<(cake::Model, Vec<fruit::Model>)> = Cake::find()
    .find_with_related(Fruit)
    .all(db)
    .await?;

批量加载​

自 0.11 开始，我们引入了 LoaderTrait 用于批量加载相关实体。

与积极加载相比，它节省了带宽（考虑一对多情况，单侧行可能重复），代价是增加一次（或在多对多情况下增加两次）数据库往返。

一对一​

使用 load_one 方法。

let fruits: Vec<fruit::Model> = Fruit::find().all(db).await?;
let cakes: Vec<Option<cake::Model>> = fruits.load_one(Cake, db).await?;

一对多​

使用 load_many 方法。

let cakes: Vec<cake::Model> = Cake::find().all(db).await?;
let fruits: Vec<Vec<fruit::Model>> = cakes.load_many(Fruit, db).await?;

多对多​

使用 load_many_to_many 方法。您必须提供交叉实体。

let cakes: Vec<cake::Model> = Cake::find().all(db).await?;
let fillings: Vec<Vec<filling::Model>> = cakes.load_many_to_many(Filling, CakeFilling, db).await?;

分页结果​

将任何 SeaORM 选择转换为具有自定义页面大小的 paginator。

use sea_orm::{entity::*, query::*, tests_cfg::cake};
let mut cake_pages = cake::Entity::find()
    .order_by_asc(cake::Column::Id) // 按 ID 升序排列
    .paginate(db, 50); // 每页 50 条

while let Some(cakes) = cake_pages.fetch_and_next().await? {
    // 对 cakes: Vec<cake::Model> 执行某些操作
}

游标分页​

如果您希望根据主键等列进行分页，请使用游标分页。

use sea_orm::{entity::*, query::*, tests_cfg::cake};
// 创建一个按 `cake`.`id` 排序的游标
let mut cursor = cake::Entity::find().cursor_by(cake::Column::Id);

// 按 `cake`.`id` > 1 和 `cake`.`id` < 100 过滤分页结果
cursor.after(1).before(100);

// 获取前 10 行（按 `cake`.`id` 升序排列）
for cake in cursor.first(10).all(db).await? {
    // 对 cake: cake::Model 执行某些操作
}

// 获取最后 10 行（按 `cake`.`id` 降序排列，但行以升序返回）
for cake in cursor.last(10).all(db).await? {
    // 对 cake: cake::Model 执行某些操作
}

基于复合主键的行分页也是可用的。

use sea_orm::{entity::*, query::*, tests_cfg::cake_filling};
let rows = cake_filling::Entity::find()
    .cursor_by((cake_filling::Column::CakeId, cake_filling::Column::FillingId)) // 以复合主键排序
    .after((0, 1))
    .before((10, 11))
    .first(3)
    .all(&db)
    .await?;
