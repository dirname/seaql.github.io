实体结构

让我们看一个简单的 Cake 实体。

use sea_orm::entity::prelude::*;

#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "cake")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub name: String,
}

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {
    #[sea_orm(has_many = "super::fruit::Entity")]
    Fruit,
}

impl Related<super::fruit::Entity> for Entity {
    fn to() -> RelationDef {
        Relation::Fruit.def()
    }
}

impl ActiveModelBehavior for ActiveModel {}
info

即使 Relation 枚举或 ActiveModelBehavior impl 块是空的，也请不要删除它们。

实体​

DeriveEntityModel 宏完成了定义一个 Entity 的所有繁重工作，并关联了 Model、Column 和 PrimaryKey。

表名​

table_name 属性指定数据库中对应的表。 可选地，您也可以通过 schema_name 指定数据库模式或数据库名称。

列名​

默认情况下，所有列名都假定为蛇形命名（snake_case）。您可以通过指定 rename_all 属性来覆盖该行为。

#[sea_orm(rename_all = "camelCase")]
pub struct Model { ... }
这里可以找到有效的 `rename_all` 属性值列表
  • camelCase
  • kebab-case
  • mixed_case
  • SCREAMING_SNAKE_CASE
  • snake_case
  • title_case
  • UPPERCASE
  • lowercase
  • SCREAMING-KEBAB-CASE
  • PascalCase

列​

列名​

您可以通过指定 column_name 属性覆盖列名。

#[derive(DeriveEntityModel)]
#[sea_orm(table_name = "user", rename_all = "camelCase")]
pub struct Model {
    #[sea_orm(primary_key)]
    id: i32,
    first_name: String, // firstName
    #[sea_orm(column_name = "lAsTnAmE")]
    last_name: String, // lAsTnAmE
}

您可以通过 column_type 属性覆盖 Rust 类型与 ColumnType 之间的默认映射。

#[sea_orm(column_type = "Text")]
pub name: String

如果您需要将 JSON 字段反序列化为结构体，您需要为其派生 FromJsonQueryResult。

#[derive(Clone, Debug, PartialEq, Eq, DeriveEntityModel)]
#[sea_orm(table_name = "json_struct")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    // 在 `serde_json::Value` 中定义的 JSON 列
    pub json: Json,
    // 在自定义结构中定义的 JSON 列
    pub json_value: KeyValue,
    pub json_value_opt: Option<KeyValue>,
}

// 自定义结构必须派生 `FromJsonQueryResult`、`Serialize` 和 `Deserialize`
#[derive(Clone, Debug, PartialEq, Eq, Serialize, Deserialize, FromJsonQueryResult)]
pub struct KeyValue {
    pub id: i32,
    pub name: String,
    pub price: f32,
    pub notes: Option<String>,
}
info

数组数据类型是 PostgreSQL 独有的功能。您可以定义已经由 SeaORM 支持的类型的向量。

#[derive(Clone, Debug, PartialEq, DeriveEntityModel)]
#[sea_orm(table_name = "collection")]
pub struct Model {
    #[sea_orm(primary_key)]
    pub id: i32,
    pub integers: Vec<i32>,
    pub integers_opt: Option<Vec<i32>>,
    pub floats: Vec<f32>,
    pub doubles: Vec<f64>,
    pub strings: Vec<String>,
}

附加属性​

您可以为列添加额外的属性 default_value、unique、indexed 和 nullable。

如果您为可选属性指定了自定义 column_type，则必须同时指定 nullable。

#[sea_orm(column_type = "Text", default_value = "Sam", unique, indexed, nullable)]
pub name: Option<String>

在选择和保存时转换列类型​

如果您需要将某一列选择为一种类型，但在数据库中以另一种类型保存，可以指定 select_as 和 save_as 属性以执行转换。一个典型用例是在 Rust 中选择 citext（不区分大小写文本）类型的列作为 String，并将其保存为 citext。应该如下定义模型字段：

#[sea_orm(select_as = "text", save_as = "citext")]
pub case_insensitive_text: String

忽略属性​

如果您希望忽略某个特定的模型属性，使其与数据库列不映射，可以使用 ignore 注解。

#[sea_orm(ignore)]
pub ignore_me: String

主键​

使用 primary_key 属性将某一列标记为主键。

#[sea_orm(primary_key)]
pub id: i32

自动递增​

默认情况下，primary_key 列隐含 auto_increment。通过指定 false 可以覆盖此行为。

#[sea_orm(primary_key, auto_increment = false)]
pub id: i32

复合主键​

这通常发生在连接表中，其中一个两列组合用作主键。只需对多个列添加注解即可定义复合主键。复合主键的 auto_increment 为 false。

主键的最大元数为 12。

pub struct Model {
    #[sea_orm(primary_key)]
    pub cake_id: i32,
    #[sea_orm(primary_key)]
    pub fruit_id: i32,
}

关系​

DeriveRelation 是一个宏，用于帮助您实现 RelationTrait。

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {
    #[sea_orm(has_many = "super::fruit::Entity")]
    Fruit,
}

如果没有关系，只需写：

#[derive(Copy, Clone, Debug, EnumIter, DeriveRelation)]
pub enum Relation {}

Related trait 连接实体，使您可以构建选择两个实体的查询。

在 Relation 章节中了解更多关于关系的信息。

活动模型行为​

针对 ActiveModel 的不同操作的钩子。例如，您可以执行自定义验证逻辑或触发副作用。在一个事务中，您甚至可以在操作完成后中止它，以防止其保存到数据库中。

#[async_trait]
impl ActiveModelBehavior for ActiveModel {
    /// 创建一个具有默认值的新 ActiveModel。也用于 `Default::default()`。
    fn new() -> Self {
        Self {
            uuid: Set(Uuid::new_v4()),
            ..ActiveModelTrait::default()
        }
    }

    /// 在插入/更新之前触发
    async fn before_save<C>(self, db: &C, insert: bool) -> Result<Self, DbErr>
    where
        C: ConnectionTrait,
    {
        if self.price.as_ref() <= &0.0 {
            Err(DbErr::Custom(format!(
                "[before_save] 无效价格, 插入: {}",
                insert
            )))
        } else {
            Ok(self)
        }
    }

    /// 在插入/更新之后触发
    async fn after_save<C>(model: Model, db: &C, insert: bool) -> Result<Model, DbErr>
    where
        C: ConnectionTrait,
    {
        Ok(model)
    }

    /// 在删除之前触发
    async fn before_delete<C>(self, db: &C) -> Result<Self, DbErr>
    where
        C: ConnectionTrait,
    {
        Ok(self)
    }

    /// 在删除之后触发
    async fn after_delete<C>(self, db: &C) -> Result<Self, DbErr>
    where
        C: ConnectionTrait,
    {
        Ok(self)
    }
}

如果不需要自定义，只需写：

impl ActiveModelBehavior for ActiveModel {}
