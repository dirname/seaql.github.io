SeaORM 概念

在 SeaORM 中，一个包含多个表的数据库称为 Schema。

每个表对应于 SeaORM 中的一个 Entity，它帮助你对相关表执行 CRUD（创建、读取、更新和删除）操作。

Entity 特征提供了一套 API，以便你在运行时检查其属性（Column、Relation 和 PrimaryKey）。

每个表都有多个列，这些列被称为 attribute。

这些属性及其值被分组在一个 Rust 结构体（一个 Model）中，以便于你对其进行操作。

然而，Model 仅用于读取操作。要执行插入、更新或删除，你需要使用 ActiveModel，它在每个属性上附加了元数据。

最后，SeaORM 中没有单例（全局上下文）。应用程序代码负责管理 DatabaseConnection 的所有权。我们提供了一些与包括 Rocket、Actix、axum 和 poem 的 Web 框架的集成示例，帮助你快速入门。
