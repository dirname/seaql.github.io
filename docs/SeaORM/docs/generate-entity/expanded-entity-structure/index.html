扩展实体结构

SeaORM 是动态的，这意味着您可以在运行时灵活地配置所有内容。如果您想知道 DeriveEntityModel 扩展为什么，可以继续阅读。否则，您可以暂时跳过这部分。

可以使用 sea-orm-cli 通过 --expanded-format 选项生成扩展的实体格式。

让我们逐步了解扩展的 Cake 实体的各个部分。

实体​

通过实现 EntityTrait，您可以在给定的表上执行 CRUD 操作。

#[derive(Copy, Clone, Default, Debug, DeriveEntity)]
pub struct Entity;

impl EntityName for Entity {
    fn schema_name(&self) -> Option<&str> {
        None
    }

    fn table_name(&self) -> &str {
        "cake"
    }
}

列​

代表此表中所有列的枚举。

#[derive(Copy, Clone, Debug, EnumIter, DeriveColumn)]
pub enum Column {
    Id,
    Name,
}

impl ColumnTrait for Column {
    type EntityName = Entity;

    fn def(&self) -> ColumnDef {
        match self {
            Self::Id => ColumnType::Integer.def(),
            Self::Name => ColumnType::String(None).def(),
        }
    }
}

所有列名假定为蛇形命名。您可以通过指定 column_name 属性来覆盖列名。

pub enum Column {
    Id,      // 映射到 SQL 中的 "id"
    Name,    // 映射到 SQL 中的 "name"
    #[sea_orm(column_name = "create_at")]
    CreateAt // 映射到 SQL 中的 "create_at"
}

要指定每个列的数据类型，可以使用 ColumnType 枚举。

其他属性​

  • 默认值
  • 唯一
  • 索引
  • 可空
ColumnType::String(None).def().default_value("Sam").unique().indexed().nullable()

在选择和保存时转换列类型​

如果您需要将某一列以一种类型选择，但以另一种类型保存到数据库中，可以重写 select_as 和 save_as 方法以执行类型转换。一个典型的使用案例是选择类型为 citext（不区分大小写的文本）的列作为 Rust 中的 String 并将其保存到数据库中作为 citext。应如下所示重写 ColumnTrait 的方法：

use sea_orm::sea_query::{Expr, SimpleExpr, Alias}

impl ColumnTrait for Column {
    // 省略的部分...

    /// 用于选择语句的列表达式转换。
    fn select_as(&self, expr: Expr) -> SimpleExpr {
        match self {
            Column::CaseInsensitiveText => expr.cast_as(Alias::new("text")),
            _ => self.select_enum_as(expr),
        }
    }

    /// 将列的值转换为适用于数据库存储的正确类型。
    fn save_as(&self, val: Expr) -> SimpleExpr {
        match self {
            Column::CaseInsensitiveText => val.cast_as(Alias::new("citext")),
            _ => self.save_enum_as(val),
        }
    }
}

主键​

代表此表主键的枚举。复合键用具有多个变体的枚举表示。

ValueType 定义了 InsertResult 中最后插入 ID 的类型。

auto_increment 定义主键是否具有自动生成的值。

#[derive(Copy, Clone, Debug, EnumIter, DerivePrimaryKey)]
pub enum PrimaryKey {
    #[sea_orm(column_name = "id")] // 覆盖默认列名
    Id,  // 映射到 SQL 中的 "id"
}

impl PrimaryKeyTrait for PrimaryKey {
    type ValueType = i32;

    fn auto_increment() -> bool {
        true
    }
}

示例复合键

pub enum PrimaryKey {
    CakeId,
    FruitId,
}

impl PrimaryKeyTrait for PrimaryKey {
    type ValueType = (i32, i32);

    fn auto_increment() -> bool {
        false
    }
}

模型​

用于存储查询结果的 Rust 结构。

#[derive(Clone, Debug, PartialEq, Eq, DeriveModel, DeriveActiveModel)]
pub struct Model {
    pub id: i32,
    pub name: String,
}

可空属性​

如果表列是可空的，则将其包装在 Option 中。

pub struct Model {
    pub id: i32,
    pub name: Option<String>,
}

活动模型​

ActiveModel 具有其相应 Model 的所有属性，但所有属性都被包装在 ActiveValue 中。

#[derive(Clone, Debug, PartialEq)]
pub struct ActiveModel {
    pub id: ActiveValue<i32>,
    pub name: ActiveValue<Option<String>>,
}

关系​

指定与其他实体的关系。

#[derive(Copy, Clone, Debug, EnumIter)]
pub enum Relation {
    Fruit,
}

impl RelationTrait for Relation {
    fn def(&self) -> RelationDef {
        match self {
            Self::Fruit => Entity::has_many(super::fruit::Entity).into(),
        }
    }
}

相关​

定义特征边界以帮助您查询相关的实体，特别是在多对多关系中非常有用。

impl Related<super::fruit::Entity> for Entity {
    fn to() -> RelationDef {
        Relation::Fruit.def()
    }
}

impl Related<super::filling::Entity> for Entity {
    fn to() -> RelationDef {
        super::cake_filling::Relation::Filling.def()
    }

    fn via() -> Option<RelationDef> {
        Some(super::cake_filling::Relation::Cake.def().rev())
    }
}
