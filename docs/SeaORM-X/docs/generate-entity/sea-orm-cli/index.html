使用 sea-orm-cli

使用 cargo 在本地安装 sea-orm-cli。

cargo install --path "<SEA_ORM_X_ROOT>/sea-orm-x/sea-orm-cli"

配置环境​

在你的环境中设置 DATABASE_URL，或者在项目根目录下创建一个 .env 文件。指定你的数据库连接。

.env
DATABASE_URL=protocol://username:password@localhost/database

获取帮助​

在任何 CLI 命令或子命令上使用 -h 标志以获取帮助。

# 列出所有可用命令
sea-orm-cli -h

# 列出 `generate` 命令下所有可用的子命令
sea-orm-cli generate -h

# 显示如何使用 `generate entity` 子命令
sea-orm-cli generate entity -h

生成实体文件​

发现数据库中的所有表，并为每个表生成相应的 SeaORM 实体文件。

支持的数据库：

  • MSSQL
  • MySQL
  • PostgreSQL
  • SQLite

命令行选项：

  • -u / --database-url：数据库 URL（默认：环境变量中指定的 DATABASE_URL）
  • -s / --database-schema：数据库模式（默认：环境变量中指定的 DATABASE_SCHEMA）
    • 对于 MySQL 和 SQLite，此参数被忽略
    • 对于 PostgreSQL，此参数是可选的，默认值为 'public'
    • 对于 MSSQL，此参数是可选的，默认值为 'dbo'
  • -o / --output-dir：实体文件输出目录（默认：当前目录）
  • -v / --verbose：打印调试信息
  • -l / --lib：生成索引文件为 lib.rs 而不是 mod.rs
  • --include-hidden-tables：从隐藏表生成实体文件（以下划线开头的表名被隐藏，默认情况下被忽略）
  • --ignore-tables：跳过指定表的实体文件生成（默认：seaql_migrations）
  • --compact-format：生成紧凑格式的实体文件（默认：true）
  • --expanded-format：生成扩展格式的实体文件
  • --with-serde：自动派生 serde Serialize / Deserialize 特性用于实体（none, serialize, deserialize, both）（默认：none）
    • --serde-skip-deserializing-primary-key：生成主键字段标记为 #[serde(skip_deserializing)] 的实体模型
    • --serde-skip-hidden-column：生成隐藏列（列名以 _ 开头）的实体模型字段标记为 #[serde(skip)]
  • --date-time-crate：用于生成实体的日期时间 crate（chrono, time）（默认：chrono）
  • --max-connections：在连接池中初始化的最大数据库连接数（默认：1）
  • --model-extra-derives：将额外的派生宏附加到生成的模型结构
  • --model-extra-attributes：将额外的属性附加到生成的模型结构
  • --seaography：在实体中生成附加结构以进行 seaography 集成
# 将数据库 `bakery` 的实体文件生成到 `entity/src`
sea-orm-cli generate entity -u protocol://username:password@localhost/bakery -o entity/src
