# 愿景

我们希望让 Rust 成为最佳的数据工程平台，从开发到生产，提供一系列的工具。

以下只是我的一部分想象，希望我们有朝一日能够实现它们！

## 流处理的 `sed`

既然我们已经谈到了很多关于 shell 的内容，如果能有一种类似 `sed` 的工具，让我们能够使用脚本语言来过滤、转换和组合结构化的消息，那就太好了。

## 进程间通信

目前，管道是连接处理器的唯一机制，这在数据复制上较为沉重。对于一些高吞吐量的场景，共享内存可能会表现得更好。

## 主机间通信

目前跨主机通信的唯一方式是依赖于流服务器。在容器化环境中，允许处理器通过 TCP 流私密地通信可能更有意义。
