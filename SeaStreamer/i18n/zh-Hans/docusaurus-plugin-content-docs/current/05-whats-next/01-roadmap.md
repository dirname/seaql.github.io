# 发展路线图

感谢您阅读到本章节的文档。请再次允许我向您提出请求，如果您觉得这个项目有趣或有用，请给我们的[GitHub仓库](https://github.com/SeaQL/sea-streamer)点个星！您的支持对于SeaStreamer持续发展至关重要。

以下是接下来我们计划开发的几个主要组件：

## `sea-streamer-file`：文件后端

在`0.3`版本中已发布初步实现。但仍有待办事项：

1. 可恢复性：当前未实现。一种潜在的实现可能是提交到本地SQLite数据库。
2. 分片：目前它仅流向分片ZERO。
3. 验证：一个用于验证和修复SeaStreamer二进制文件的实用程序。

## `sea-streamer-redis`：Redis集群

Redis支持已在`sea-streamer`的`0.2`版本中发布！基本的流分片已经实现，但是没有集群的分片用处不大。

未来，我们希望支持Redis Cluster，目前这几乎是一个正在进行的工作。这是一个相当艰巨的任务，因为与集群一起工作时客户端需要承担更多责任。
在Redis中，分片和节点是动态的M-N映射 - 分片可以随时在节点间移动。
这让测试变得更加困难。

在Redis中，同一组的消费者共享相同的分片，即共享分片。未来，我们希望支持“独有分片”语义，其中每个消费者将尝试声明对分片的所有权，而组中的其他消费者不会介入。这模仿了Kafka的消费者组行为。
