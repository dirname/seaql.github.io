# SeaStreamer 概念

## 流媒体服务器

流媒体服务器，通过一个 URI 标识，所有生产者和消费者都可以连接到这个 URI。

### 集群

假设流媒体服务器是一个集群：它可以在多个节点上水平扩展。

## 流

一个流由一系列具有相同键的消息组成（在 Kafka 中称为“主题”）。每条消息都有一个时间戳、序列号（在 Kafka 中称为“偏移量”）、分片 ID（在 Kafka 中称为“分区编号”）以及负载。一条消息可以通过（流键、分片 ID、序列号）元组唯一标识。

### 流 URL

在 SeaStreamer 中，流是资源，可以通过包含（协议、主机、流）的 URL 访问。例如，流 URL 可能是 `kafka://streamer.sea-ql.org:12345/my_stream`。

## 消费者

流消费者订阅一个或多个流，并从集群中的一个或多个节点接收消息。

消费者可以将流倒回至任意点（由时间戳或序列号定位），并继续流式传输。

### 消费模式

有三种消费模式：

1. 实时：我们只关心最新消息，可以忽略旧数据
2. 可恢复：当消费者重新订阅时，它将从上次消费的消息处继续
3. 负载均衡：类似于可恢复模式，但多个消费者共享同一组流

## 生产者

流生产者向流媒体服务器发送消息，服务器会在集群内存储这些消息，并将它们传递给客户端。

生产者可以使用任何流键发送消息，但在 SeaStreamer 中，我们建议每个生产者绑定到特定的流键。

## 处理器

流处理器同时是消费者和生产者。它消费消息，转换它们，并产生另一个流。

SeaStreamer 致力于使开发和操作流处理器变得简单且灵活。

## 流语义

高级概念，如分片、负载均衡和事务处理，都是后端特定的，你应该阅读相关流后端的文档。
