# 流处理简介

“[将数据库翻转](https://www.confluent.io/blog/turning-the-database-inside-out-with-apache-samza/)” 是数据工程领域中一篇有影响力的文章，它促成了[Kafka](https://kafka.apache.org/)的诞生。从那时起，诸如[Redpanda](https://redpanda.com/) 和 [Redis Streams](https://redis.io/docs/manual/data-types/streams/)等实现相继出现，推动了实时数据处理生态系统的形成。

## 与事件驱动编程对比

流处理与事件驱动编程类似，都是一种旨在处理近乎实时或在事件发生时处理事件的编程范式。区分两者的一种方式可能是频率。流是高吞吐量的连续事件序列：你只需保持连接打开，等待事件的到来，而不是建立许多短生命周期的连接。

## 与批处理对比

流处理可以看作是具有极小批量大小的批处理。批量大小不一定必须是1。根据你的延迟要求和处理能力，消息可以按毫秒或秒进行微批处理。但“这个更改可能需要长达24小时才能反映出来”的时代已经一去不复返了！

## 为什么选择Rust？

我们希望建立最佳的流处理平台，在这里Rust的独特特性真正得以展现：

### 多线程异步

与其他语言不同，Rust的异步执行是多线程的。它允许你根据需要扩展进程，使用尽可能多的线程来充分利用CPU，以达到最大并发性。**你无需设置外部队列系统**！

### 可预测的延迟

作为一种没有垃圾回收的语言，不会在任意时间点触发垃圾收集器导致抖动。当你有一个长流水线时，这些抖动倾向于向下传播并放大。Rust并不自动低延迟，但你将有一个良好的起点。

### 自包含

与其他语言不同，推荐的Rust程序打包方式是静态链接所有内容到一个可执行文件——通常仅几兆字节大小。并且无需安装或预热，它可以立即启动，这对于流处理是一个额外的好处。

### 资源使用低

像其他编译语言一样，Rust的内存使用明显少于基于虚拟机的语言。而且，由于无需即时编译（JIT），Rust也具有更低的CPU开销。

### 长期运行安全

再次，没有GC，Rust程序不太容易出现“数天内缓慢的内存膨胀”（技术上，这不是泄漏）。内存溢出崩溃的风险较低，因此你不必“每周重启一次进程”。尽管如此，你仍然需要小心堆分配。

### 生态系统

最后，Rust拥有丰富的异步编程库生态系统：基于异步IO构建的网络库、无锁通道和其他数据结构，使异步编程既舒适又有趣。
