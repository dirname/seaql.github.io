# 教程与示例

如果你更喜欢逐步的教程来学习如何使用 SeaORM，你可以查看我们的[SeaORM 教程](https://www.sea-ql.org/sea-orm-tutorial/)。

最好先对基本概念有所了解，所以让我们继续。

如果你非常急切，想要一些现成可用的东西，SeaQL 维护了一套由社区贡献的官方示例（我们欢迎更多的！）：

- [Actix v4 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/actix_example)
- [Axum 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/axum_example)
- [GraphQL 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/graphql_example)
- [jsonrpsee 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/jsonrpsee_example)
- [Poem 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/poem_example)
- [Rocket 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/rocket_example)
- [Salvo 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/salvo_example)
- [Tonic 示例](https://github.com/SeaQL/sea-orm/tree/master/examples/tonic_example)

如果你正在构建一个需要大量进行嵌套关系选择的 Web API。考虑使用 GraphQL 服务器。[seaography](https://github.com/SeaQL/seaography) 是一个使用 SeaORM 实体构建 GraphQL 解析器的 GraphQL 框架。有了 GraphQL 解析器，上述的嵌套关系选择变得直接而简单。查看"[开始使用 seaography](https://www.sea-ql.org/blog/2022-09-27-getting-started-with-seaography/#query-data-via-graphql)"以了解更多。

你也可以查看 [SeaORM 烹饪书](https://www.sea-ql.org/sea-orm-cookbook/)，以获取常见问题和 SeaORM 的最佳实践。
